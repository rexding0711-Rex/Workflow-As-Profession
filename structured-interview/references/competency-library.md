# 通用能力模型库

## 技术能力

### 系统设计
- **初级**：能设计单个模块的架构
- **中级**：能设计跨模块的系统交互，考虑扩展性
- **高级**：能设计分布式系统，考虑 CAP、一致性、性能瓶颈
- **面试信号**：主动讨论 trade-off（一致性 vs 可用性、性能 vs 可维护性）

### 编码能力
- **初级**：能写出正确、可运行的代码
- **中级**：代码可读、可测试、遵循最佳实践
- **高级**：能设计 API/库接口，考虑错误处理、边界条件
- **面试信号**：写代码时自言自语地解释思路，测试用例先行

### 问题排查
- **初级**：能根据文档定位问题
- **中级**：能从日志/监控中定位根因
- **高级**：能建立排查框架，预防同类问题复发
- **面试信号**：先问"什么现象？有没有最近的变更？"

## 产品能力

### 用户洞察
- **初级**：能执行用户调研
- **中级**：能从数据中提炼用户需求
- **高级**：能发现用户自己都没意识到的需求
- **面试信号**：引用具体的用户故事而非抽象需求

### 优先级判断
- **初级**：能按模板排优先级
- **中级**：能用 ICE/RICE 框架量化
- **高级**：能在信息不全时做出"对的赌注"
- **面试信号**：主动讨论"为什么不做 X"

## 通用能力

### 沟通协作
- **初级**：能清晰表达自己的观点
- **中级**：能推动跨团队协作
- **高级**：能在冲突中找到共识，影响无汇报关系的同事
- **面试信号**：使用 STAR 描述具体协作案例

### 领导力
- **初级**：能带领小型项目
- **中级**：能带领 3-5 人团队交付
- **高级**：能培养 leader，建立团队文化
- **面试信号**：描述"你改变了什么"而非"你执行了什么"

### 成长心态
- **初级**：能接受反馈
- **中级**：主动寻求反馈并改进
- **高级**：能建设团队的反馈文化
- **面试信号**：主动分享失败经历和学到的东西

## 面试方法速配

| 要考察的能力 | 推荐面试方法 | 理由 |
|------------|------------|------|
| 编码能力 | Pair Programming / Take-home | 直接观察 |
| 系统设计 | Design Task | 展示架构思维 |
| 产品感觉 | Case Study | 模拟真实决策 |
| 过去行为 | STAR / BEI | 过去预测未来 |
| 潜在能力 | Situational | 测思维过程 |
| 文化匹配 | Values Interview | 看行为而非口号 |
