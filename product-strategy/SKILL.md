# Product Strategy — 产品战略与路线图

"我们要做最好的 XX 产品"不是战略。"我们选择服务 XX 人群的 YY 需求，
为此我们不做 ZZ，我们的优势是 AA"才是。战略不是你要做什么，
是你选择**不做什么**。

## 何时触发

用户提及以下内容时触发：
- "我们的产品方向是什么？"
- "帮我做一个 Q3 的产品路线图"
- "竞品刚上线了一个和我们一模一样的功能，怎么办？"
- "怎么判断我们的产品有没有 PMF？"
- "老板让我做明年的产品规划"
- "Product strategy and roadmap planning"

**排除：** 此技能 NOT for：
- 单功能 PRD → 用 `prd-writer`
- Backlog 管理和迭代规划 → 用 `user-story-mapping`

## 开始之前

| # | 信息 | 示例 |
|---|------|------|
| 1 | **产品现状** | "B 轮 SaaS，500 家付费客户，ARR ¥20M" |
| 2 | **目标市场** | "中国中型企业的协同办公" |
| 3 | **核心竞品** | "飞书、钉钉、企业微信" |
| 4 | **战略问题** | "增长停滞 / 大厂也进来了 / 不知道下一步做啥" |

## 产品战略金字塔

```
       愿景 — 5 年后世界因为你的产品变成什么样？
      /    \
   战略 — 你在哪玩、怎么赢？（2-3 年）
  /    |    \
目标 — OKR/KPI（季度/年度）
|    |    |    |
路线图 — 主题 → Epic → Story（季度/月度）
```

## 战略选择框架

### 在哪玩 × 怎么赢

| 选择 | 含义 | 示例 |
|------|------|------|
| **在哪玩** | 目标市场 + 目标用户 + 核心需求 | "中国中型企业，100-500 人，协同办公" |
| **怎么赢** | 差异化定位 + 核心优势 | "专注异步沟通 + 知识沉淀，而非实时聊天" |

### 差异化矩阵

```
              竞品做得好          竞品做得差
你的       ┌─────────────────┬─────────────────┐
强项       │ 正面竞争（难）    │ 🏆 核心差异（打！）│
          ├─────────────────┼─────────────────┤
你的       │ ⚠️ 短板（补！）   │ 都不行（不重要）   │
弱项       └─────────────────┴─────────────────┘
```

**找到你的 🏆 象限：** 竞品做得差但你做得好的地方 = 你的所有弹药应该集中在这。

### 大厂来了怎么办？

| 大厂的弱点 | 你的机会 |
|-----------|---------|
| 通用产品，不管细分 | 聚焦一个细分场景，做到极致 |
| 反应慢，更新周期长 | 快速迭代，每周发版 |
| 客服是机器人 | 给客户一个微信，什么问题都秒回 |
| 不能定制 | 给前 10 个客户做白手套服务 |

## PMF（产品-市场匹配）评估

### Sean Ellis 测试

问用户："如果你不能再使用这个产品，你会有什么感受？"

| 回答 | 含义 |
|------|------|
| "非常失望" ≥40% | ✅ 有 PMF 了 |
| "非常失望" 25-40% | ⚠️ 接近 PMF，还没完全到 |
| "非常失望" <25% | ❌ 还没 PMF |

### PMF 信号清单

| 信号 | 说明 |
|------|------|
| 自然增长 >30% | 用户自己找上门，不是全靠投放 |
| NPS >40 | 用户愿意推荐 |
| 客户留存 >90%（年） | 用户离不开 |
| Sales cycle 在缩短 | 越卖越容易 |
| 用户在主动安利 | Slack/微信群里有人推荐 |

## 路线图构建

### Now-Next-Later 框架

替代传统的"时间轴路线图"（Q1 做 A、Q2 做 B），用"现在-下一个-未来"：

```
现在（Now）— 已经在做，本季度交付
  └─ 3-5 个主题

下一个（Next）— 初步验证过的，下一季度启动
  └─ 3-5 个主题

未来（Later）— 方向确定了但还没验证完
  └─ 3-10 个想法
```

### 为什么不用时间轴路线图

- 时间轴给 stakeholders 一种"承诺"的错觉
- 实际上三个月后的预测准确率 <30%
- Now-Next-Later 诚实：告诉团队现在的焦点是什么，下一步可能会做什么

### 路线图主题

路线图的单元不是"功能"，是"主题"：

| ❌ 功能列表 | ✅ 主题 |
|-----------|--------|
| "做分享按钮" | "提升内容传播"（本月） |
| "做暗黑模式" | "提升使用体验"（本季度） |
| "做多语言" | "全球化准备"（下季度） |

主题说明了你为什么要做这些功能，让工程师有上下文去提出更好的实现方案。

## 输出模板

```markdown
# [产品名] — 产品战略

## 愿景
[5 年后，你的产品让世界发生了什么变化？]

## 战略定位
- **在哪玩：** [市场 + 用户 + 需求]
- **怎么赢：** [差异化 + 核心优势]
- **不做什么：** [明确的边界]

## 竞争格局
| 竞品 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|-----------|
| [A] | [优势] | [弱点] | [机会] |

## PMF 状态
[当前 PMF 证据 + 差距]

## 路线图（Now-Next-Later）

### Now（本季度）
| 主题 | 要解决的问题 | 成功指标 |
|------|-----------|---------|
| [主题1] | [问题] | [指标] |

### Next（下季度）
| 主题 | 要验证的假设 |
|------|-----------|
| [主题] | [假设] |

### Later（未来）
| 想法 | 触发条件 |
|------|---------|
| [想法] | [如XX数据达到YY] |

## 关键假设与风险
| 假设 | 如果不成立 |
|------|----------|
| [假设] | [Plan B] |
```

## 质量原则

1. **战略 = 选择不做什么** — 如果你的战略里没有明确的 No，说明还没想清楚
2. **路线图 ≠ 功能列表** — 用主题替代功能，让团队知道"为什么"
3. **PMF 不是 0/1** — 是连续的，而且可以丢失（Scale 之后 PMF 可能变弱）
4. **大厂来了不意味着你完了** — 大厂做的是"所有人的 80% 需求"，你做的是"一小群人的 100% 需求"
5. **路线图是活的** — 每季度 review 一次，基于新数据和反馈调整 Next 和 Later

## 使用示例

**用户输入：**
> 我们做了两年的项目管理 SaaS，小团队用得挺好，但增长停了。现在飞书也做了项目管理功能。怎么办？

**AI 执行过程：**
1. 差异化矩阵分析：飞书项目管理=通用，竞品做得差的地方是什么？
2. 发现可能机会：飞书不太管"外包团队管理"、"甲方乙方协作"这类跨公司场景
3. 建议：把聚焦从小团队内部协作 → 跨公司项目协作（甲方+乙方+丙方在一个项目里）
4. 这个市场飞书看不上，但养活一家 SaaS 公司够了
5. 输出战略定位 + Now-Next-Later 路线图
