# JD Writer — 职位描述撰写

撰写专业、包容、高转化的职位描述。好的 JD 不是需求清单，而是一封给候选人的"情书"——告诉他为什么这个岗位值得他来。

## 何时触发

用户提及以下内容时触发：
- "帮我写一个 JD" / "帮我写一份岗位描述"
- "Write a job description for..."
- "招一个高级前端，帮我写 JD"
- "这个 JD 帮我优化一下"
- "帮我把 JD 改成 LinkedIn 版本"

**排除：** 此技能 NOT for：
- 面试问题设计 → 用 `structured-interview`
- 薪酬谈判策略 → 不属于 JD 范畴
- 录用决策 → 不属于 JD 范畴

## 开始之前

在开始撰写 JD 前，收集以下信息。如用户未提供，主动逐项询问（一次最多 2 个问题）：

### 必收集项

| # | 信息 | 示例 |
|---|------|------|
| 1 | **岗位名称** | "高级前端工程师" |
| 2 | **团队/部门** | "增长团队，10 人，汇报给 CTO" |
| 3 | **核心职责** (3-5 条) | "负责营销后台系统搭建" |
| 4 | **必备技能** | "React 3 年+、TypeScript" |
| 5 | **工作地点 + 模式** | "上海，混合办公" |

### 加分收集项

| # | 信息 | 示例 |
|---|------|------|
| 6 | 薪酬范围 | "30K-45K × 15 薪" |
| 7 | 公司一句话介绍 | "B 轮 SaaS 公司，50 人" |
| 8 | 面试流程 | "3 轮：技术面→跨部门→HR" |
| 9 | 目标渠道 | "LinkedIn + Boss 直聘" |

## 工作模式

### 模式 A：从零撰写
用户给出岗位名 + 大致背景，你生成完整 JD。适用于新岗位。

### 模式 B：JD 优化
用户给出已有 JD，你提供优化建议 + 改写版本。检查包容性语言、SEO 关键词、结构完整性。

### 模式 C：渠道适配
用户给出一个版本的 JD，你生成适配不同招聘渠道的版本（LinkedIn / 猎聘 / Boss 直聘 / Indeed）。

## JD 标准结构

每个 JD 必须包含以下模块（按顺序）：

```
1. 岗位标题（含级别 + 方向）
2. 一句话亮点（Why this role matters）
3. 岗位概述（3-5 句话，说清楚这个岗位做什么、为什么重要）
4. 核心职责（5-8 条，action-oriented）
5. 必备资格（Must-have，5-8 条）
6. 加分项（Nice-to-have，3-5 条）
7. 薪酬范围（如有）
8. 面试流程（如有）
9. 关于我们（公司介绍，2-3 句话）
10. 平等雇佣声明
```

## 包容性语言检查

撰写完成后，对 JD 做以下检查并标注修改建议：

| 检查项 | ❌ 避免 | ✅ 推荐 |
|--------|--------|--------|
| 性别中立 | "他/她"、"manpower" | "他们"、"workforce" |
| 年龄中立 | "年轻有活力"、"应届生优先" | "经验不限"、"欢迎各阶段候选人" |
| 能力中立 | "正常视力"、"身体健康" | 描述具体工作能力要求 |
| 不必要的年限 | "10 年以上经验" | "精通 XXX，能独立负责..." |
| 文化契合度替代 | "culture fit" | "culture add"（你能带来什么新东西） |

如检测到 3+ 个问题，在输出末尾独立列出 **包容性改进建议**。

## 渠道格式适配

不同渠道的 JD 长度和风格差异：

| 渠道 | 长度 | 风格 | 特殊字段 |
|------|------|------|---------|
| **LinkedIn** | 600-1200 词 | 专业、结构化 | #hashtags, 技能标签 |
| **Boss 直聘** | 200-500 字 | 直接、接地气 | 必须含"聊一聊"引导语 |
| **猎聘** | 500-1000 字 | 中高端、稳重 | 强调发展空间 |
| **Indeed** | 300-700 词 | SEO 友好 | 关键词密度要高 |

## 输出模板

```markdown
# [岗位名称]

> [一句话亮点 — 这个岗位为什么重要/有趣]

## 岗位概述
[3-5 句话描述岗位的核心使命、所在团队、对公司的价值]

## 核心职责
1. [职责 1 — 用动词开头，描述产出而非活动]
2. [职责 2]
3. [职责 3]
...

## 必备资格
- [技能/经验 1]
- [技能/经验 2]
...
- 工作模式：[远程/混合/现场]

## 加分项
- [加分技能/经验 1]
- [加分技能/经验 2]
...

## 薪酬与福利
- 薪酬范围：[XX-XX]（如有）
- 核心福利：[2-3 个亮点福利]

## 面试流程
1. [阶段 1]
2. [阶段 2]
3. [阶段 3]

## 关于我们
[公司一句话介绍 + 1-2 个有吸引力的信息点]

---
*[公司名] 是平等雇佣机会雇主。我们欢迎不同背景、经验和视角的候选人。*
```

## 质量原则

1. **职责描述用产出而非活动** — "负责提升转化率" 优于 "负责数据看板维护"
2. **每条职责可衡量** — 如果无法衡量完成标准，重新措辞
3. **必备资格 ≤ 8 条** — 超过说明你在招超人，不是现实的人
4. **加分项不含歧视性暗示** — "名校毕业" 改为 "计算机相关专业"
5. **薪酬范围尽量透明** — 如未提供，标注"视经验而定"并建议用户补充
6. **一句话亮点必须在 JD 开头** — 拒绝平庸

## 使用示例

**用户输入：**
> 帮我写一个高级后端工程师的 JD，做 SaaS 产品的，团队 5 个人，远程优先，需要 Go 和 k8s 经验。

**AI 执行过程：**
1. 确认必收集项 + 追问薪酬范围和工作地点
2. 生成模式 A：从零撰写 JD
3. 运行包容性语言检查（通过，无问题）
4. 输出 LinkedIn 版本 + Boss 直聘适配版本
5. 问用户是否需要调整
