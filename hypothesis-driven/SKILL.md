# 假设驱动（Hypothesis-driven）

## 何时触发
用户需要分析问题时，不想被海量数据淹没，想快速找到方向。例如：
- "帮我分析这个问题，数据太多了"
- "先给一个初步判断，我再验证"
- "怎么快速找到问题的根因"

## 核心框架：先猜答案，再验证

### 传统方法 vs 假设驱动

| 传统方法 | 假设驱动 |
|---------|----------|
| 收集所有数据 → 分析 → 得出结论 | 先提出假设 → 收集验证数据 → 证实/证伪 |
| 数据驱动，容易信息过载 | 假设引导，聚焦关键数据 |
| 耗时、成本高、结论散 | 快速、聚焦、有方向 |

### 假设驱动的三步法

**Step 1：提出假设（基于经验和逻辑）**
- 不要等数据，先用常识/经验/行业知识猜一个答案
- 假设要具体、可证伪：不是"可能有问题"，而是"价格过高导致客户流失率上升15%"
- 同时提出2-3个备选假设（避免锚定偏见）

**Step 2：设计验证（找到关键数据）**
- 每个假设需要1-2个"关键数据点"来验证
- 问自己："如果我的假设是对的，我会在数据中看到什么？"
- 80/20：验证假设只需要20%的数据，不是100%

**Step 3：证实/证伪/调整**
- 数据支持假设 → 深化分析，向下拆解
- 数据不支持假设 → 转向备选假设，或提出新假设
- 数据部分支持 → 修正假设，更精确化

### 假设的质量标准

**好假设的特征**：
- 具体：包含数字、时间、因果关系
- 可证伪：存在数据可以证明它错
- 有影响力：如果对了，能解决问题的大部分
- 快速可验证：能在1-2天内找到验证数据

**坏假设的特征**：
- 模糊："可能有问题"
- 循环："因为业绩差，所以业绩差"
- 不可验证："员工士气低落"（没有量化数据）
- 无关：即使对了，对大问题影响很小

### 假设树（Hypothesis Tree）

将假设与Issue Tree结合：
```
问题：销售额下降
├── 假设A：竞品降价（影响度40%）
│   ├── 验证：竞品价格变化？
│   └── 数据：价格对比表
├── 假设B：核心客户流失（影响度35%）
│   ├── 验证：Top 20客户流失率？
│   └── 数据：客户交易记录
├── 假设C：新产品上市延迟（影响度15%）
│   ├── 验证：产品上市时间表？
│   └── 数据：研发进度
└── 假设D：销售团队变动（影响度10%）
    ├── 验证：人员变动率？
    └── 数据：HR记录
```

## 输出模板

```markdown
# [问题] 假设驱动分析

## 1. 问题定义
- 核心问题：[一句话]
- 问题类型：[Why/How/What]
- 现有信息：[已知的事实]
- 信息缺口：[还不知道的]

## 2. 假设生成（基于经验/逻辑）
- 假设1：[具体、可证伪的陈述]
  - 为什么猜这个：[经验/数据/逻辑]
  - 影响度：[如果对了，能解决多少问题？]
- 假设2：[备选]
  - 为什么猜这个：
  - 影响度：
- 假设3：[保守/相反]
  - 为什么猜这个：
  - 影响度：

## 3. 验证设计（80/20数据）
- 假设1验证：[需要1-2个关键数据点]
- 假设2验证：[需要1-2个关键数据点]
- 假设3验证：[需要1-2个关键数据点]
- 数据可得性：[哪些数据能拿到？哪些拿不到？]
- 验证时间：[每个假设需要多久验证？]

## 4. 验证结果
- 假设1：[证实/证伪/部分证实]
  - 数据证据：[什么数据支持/反对？]
  - 修正后假设：[如果需要修正]
- 假设2：[...]
- 假设3：[...]

## 5. 结论与下一步
- 最可能假设：[哪个假设被最强支持？]
- 置信度：[高/中/低]
- 下一步：[基于这个假设，做什么？]
- 需要深化的子问题：[拆到下一层]
```

## 经典案例

- **麦肯锡"客户流失"**：假设"Top 20客户流失"→验证"Top 20交易记录"→发现80%流失来自3个客户→深入分析这3个客户
- **BCG"成本优化"**：假设"物流成本过高"→验证"运输单价 vs 行业基准"→发现是路线规划问题→优化路线
- **贝恩"定价策略"**：假设"价格弹性高，降价能提升收入"→验证"历史价格变化与销量关系"→发现弹性低，维持价格

## 使用示例

**用户输入**：
> "我们客户投诉率上升了50%，帮我分析原因"

**AI执行**：
1. 提出3个假设：产品质量下降/服务响应变慢/竞品服务更好
2. 设计验证：产品退货率/客服响应时间/竞品NPS对比
3. 快速验证：假设"服务响应变慢"最可能（投诉集中在售后）
4. 深化：拆到客服流程、人员配置、系统效率
