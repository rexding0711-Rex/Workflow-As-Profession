# 领导力与决策 Skill

## 何时触发

用户请求：
- 做重大商业/管理决策
- 提升领导力或团队管理能力
- 处理危机或复杂局面
- 设计决策框架或复盘机制
- "这个决策怎么做" / "怎么带团队穿越周期"

## 核心框架：决策质量 + 领导力层级 + 复盘系统

### 步骤1：决策质量评估

好决策 ≠ 好结果。决策质量取决于：
- 决策过程是否系统
- 信息是否充分
- 偏见是否被识别
- 是否考虑了反事实

**决策质量四维度**：
1. **信息质量**：是否获取了关键信息？是否遗漏了重要信息？
2. **分析质量**：是否使用了合适的框架？推理是否严密？
3. **偏见控制**：是否识别了认知偏见？是否做了对冲？
4. **执行可行性**：是否考虑了执行约束？团队能力是否匹配？

**常见决策偏见**：
- 确认偏见：只看支持自己观点的信息
- 沉没成本：因为已经投入太多，继续错误决策
- 锚定效应：过度依赖第一个信息
- 幸存者偏差：只看成功案例
- 过度自信：高估自己的预测能力
- 群体思维：为和谐而压制异议

**对抗偏见工具**：
- 红队机制：指定一个人专门挑毛病
- 事前验尸："假设这个决策失败了，为什么？"
- 反事实思考："如果当初没做这个选择，现在会怎样？"
- 基础概率：考虑同类决策的历史成功率

### 步骤2：决策类型与框架

**类型1：战略决策（不可逆、影响大）**
- 框架：预期价值 + 情景分析 + 实物期权
- 原则：慢决策，多论证，留退路
- 工具：决策树、情景规划、战争模拟

**类型2：战术决策（可逆、影响中）**
- 框架：快速迭代 + 数据反馈
- 原则：快决策，小实验，快速调整
- 工具：A/B测试、MVP、PDCA

**类型3：操作决策（高频、影响小）**
- 框架：标准化 + 授权
- 原则：建流程，少干预，自动化
- 工具：SOP、规则引擎、自动化

**决策授权原则**：
- 一线有信息但无权力 → 低质量决策
- 高层有权力但无信息 → 慢决策
- 最佳：信息在哪里，决策权就在哪里（需匹配能力）

### 步骤3：领导力层级（五级领导力）

```
Level 5: 卓越领导者（谦逊+意志）
Level 4: 有效领导者（组织绩效）
Level 3: 胜任管理者（组织能力）
Level 2: 贡献团队成员（团队贡献）
Level 1: 高能力个人（个人产出）
```

**领导力本质**：不是职位，是影响力。影响力来源：
- **专业权威**：你确实懂
- **品格信任**：你值得信任
- **愿景感召**：你描绘的未来值得追随
- **关系连接**：你关心团队成员
- **资源赋能**：你能给团队带来资源

### 步骤4：危机管理

危机类型：
- **生存危机**：资金链断裂、核心团队出走、关键客户流失
- **声誉危机**：产品质量、公关事件、创始人丑闻
- **外部危机**：政策变化、经济危机、竞争突变

**危机管理四步法**：
1. **止损**：立即控制损害范围，防止恶化
2. **诊断**：根因分析（不是表面原因）
3. **应对**：制定并执行解决方案
4. **复盘**：从中学习，防止再次发生

**危机沟通原则**：
- 快：第一时间回应，不要等完美方案
- 诚：承认问题，不要狡辩
- 行：给出具体行动，不是承诺
- 跟：持续更新进展，不要沉默

### 步骤5：决策复盘系统

**复盘模板**：

```markdown
# 决策复盘：[决策名称]

## 决策背景
- 时间：[日期]
- 决策人：[谁]
- 决策内容：[做了什么决定]
- 关键信息：[当时已知的信息]
- 备选方案：[考虑过哪些替代方案]

## 决策过程评估
- 信息充分度：[高/中/低]
- 分析质量：[高/中/低]
- 偏见控制：[是否识别了偏见]
- 参与人员：[谁参与了决策]

## 结果
- 实际结果：[发生了什么]
- 预期结果：[原本期望什么]
- 偏差分析：[为什么有偏差]

## 学习
- 做得对：[应该保留的做法]
- 可以更好：[下次如何改进]
- 模式识别：[这是否是系统性问题]

## 行动
- 立即调整：[如果还在执行中]
- 机制改进：[防止类似问题]
- 知识沉淀：[写入团队知识库]
```

**复盘频率**：
- 重大决策：1周后快速复盘，3个月后深度复盘
- 常规决策：季度批量复盘
- 危机事件：危机结束后立即复盘

## 领导力案例库

- **乔布斯**：被苹果开除后的复盘→回归后的聚焦
- **贝佐斯**："Day 1"文化 + 长期主义决策
- **任正非**：华为的"冬天"预警 + 备胎计划
- **巴菲特**：投资决策日志 + 反偏见机制
- **张一鸣**：Context not Control + 延迟满足
- **稻盛和夫**：阿米巴经营 + 现场主义

## 使用示例

**用户输入**：
> "我们公司正在考虑要不要收购一个小团队，这个决策怎么做？"

**AI执行**：
1. 分类：战略决策，不可逆，影响大
2. 设计信息收集清单：财务、技术、文化、法律
3. 列出备选方案：收购/合作/自建/不进入
4. 偏见检查：是否被FOMO驱动？是否过度乐观协同效应？
5. 给出决策框架和推荐
6. 安排复盘时间表
