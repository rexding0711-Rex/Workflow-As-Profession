# 跟进方案

## 跟进时间线

| 阶段 | 时间 | 方式 | 语气 | 内容 | 接收人 |
|------|------|------|------|------|--------|
| **首次跟进** | 截止前2天 | 即时消息 | 温和 | 确认进度、提供帮助 | 责任人 |
| **二次跟进** | 截止前1天 | 邮件+消息 | 正式 | 强调截止、确认完成时间 | 责任人 |
| **升级跟进** | 截止当天 | 邮件+消息 | 紧急 | 要求立即响应 | 责任人+主管 |
| **上报提醒** | 超时1天 | 邮件+电话 | 严肃 | 正式上报、要求处理 | 主管+部门负责人 |
| **最终升级** | 超时3天 | 邮件+电话+会议 | 正式 | 纳入绩效考核、记录档案 | 部门负责人+HR |

## 跟进话术模板

### 首次跟进（温和）
```
[姓名]你好，

关于[任务]，想确认一下当前进展。截止日期是[日期]，还有[N]天。

进展如何？有什么需要我协助的吗？

如有困难，随时沟通，我们一起想办法。

谢谢！
```

### 二次跟进（正式）
```
[姓名]你好，

[任务]的截止日期是[日期]，目前距离截止还有[N]天。

请确认：
1. 当前进度如何？
2. 预计能否按时完成？
3. 如有困难，请说明原因和需要的支持。

请务必在[日期]前完成。逾期未完成将影响[后果]。

请回复确认。
```

### 升级跟进（紧急）
```
[姓名]你好，

[任务]已于[日期]截止，目前尚未完成。

这是第[N]次提醒。请立即确认：
1. 为什么未完成？
2. 预计什么时候能完成？
3. 需要什么支持？

请于[具体时间]前回复，否则将升级至[上级]。

逾期将影响[具体后果，如：项目进度、绩效考核、客户交付]。
```

## 跟进原则

1. **逐步升级**：不要一上来就严肃，给人改进机会
2. **事实记录**：每次升级都记录时间、方式、内容
3. **后果明确**：让对方清楚不行动的后果
4. **留余地**：即使最终升级，也给对方解释和补救机会
5. **不人身攻击**：对事不对人，不评价人品，只评价行为
