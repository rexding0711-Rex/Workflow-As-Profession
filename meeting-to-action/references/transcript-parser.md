# 会议内容解析规则

## 议题识别

- 关键词触发："今天讨论..."、"关于..."、"议题..."、"第一个..."、"接下来..."
- 列表结构：1. 2. 3. / 一、二、三、/ 首先、其次、最后
- 时间标记："花了XX分钟讨论..."（通常表示重要议题）

## 决策识别

- 关键词："决定..."、"定下来..."、"通过了..."、"同意..."、"确认..."
- 条件句式："如果...就..."、"除非...否则..."
- 数字/日期：出现具体数字、日期、金额、人员名

## 行动项识别

- 关键词："负责..."、"跟进..."、"落实..."、"确认..."、"提交..."、"完成..."
- 人名+动词："[人名] 去..."、"[人名] 负责..."
- 时间词："本周"、"月底前"、"下周三"、"明天"
- 模糊行动项："看看"、"了解一下"、"考虑一下" → 标记为"待澄清"

## 责任人推断

| 场景 | 推断方法 | 示例 |
|------|----------|------|
| 明确指派 | 直接提到人名+负责 | "张三负责报价" |
| 默认负责人 | 议题与某人职责匹配 | 讨论采购→默认采购负责人 |
| 主持人承担 | 无人明确承担，主持人总结 | "我来跟进" |
| 待确认 | 多人模糊提及，无明确指派 | "大家看看" → 标记"待确认责任人" |

## 时间推断

| 表达 | 推断为 | 示例 |
|------|--------|------|
| "今天/马上/立刻" | 当日 | 会议当天 |
| "本周" | 会议所在周的周五 | 给缓冲 |
| "月底/月初" | 当月最后一天/次月第一天 | |
| "下周三" | 具体日期 | 精确 |
| "尽快/抽时间" | 标记为"待确认时间" | 无明确deadline |
| 无时间词 | 标记为"待确认时间" | 会后需补充 |

## 模糊内容处理

- **模糊行动项**：如"我们看看能不能..." → 转化为"调研可行性"或标记"待澄清"
- **讨论中的讨论**：会上说"这个问题我们再讨论" → 标记为"未决议题，需安排专题会"
- **口头承诺无记录**：如"行，我搞定" → 必须记录为行动项，会后确认
