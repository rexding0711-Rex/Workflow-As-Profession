# OKR 最佳实践

## 什么是好的 O

**O（目标）应该是：**
- 定性的（不是数字）
- 鼓舞人心的（读起来让人想去做）
- 有时限的（通常是季度）
- 可以独立影响（不依赖其他团队先完成）

**❌ 差的 O：** "提升系统稳定性"（太模糊）
**✅ 好的 O：** "让我们的 API 成为客户信赖的基础设施"（有画面感）

## 什么是好的 KR

**KR（关键结果）应该是：**
- 定量的（必须能数出来）
- 有挑战但可达（信心指数 5/10 = 一半把握）
- 是结果(outcome)而非任务(output)

**❌ 任务的 KR：** "完成 5 次客户访谈"（这是任务，不是结果）
**✅ 结果的 KR：** "NPS 从 35 提升到 50"（这是结果）

**❌ 没挑战的 KR：** "系统可用性保持 99.5%"
**✅ 有挑战的 KR：** "系统可用性从 99.5% 提升到 99.9%"

## 信心指数

每个 KR 标注 1-10 的信心指数：
- 1-3：太容易，不是 OKR，是任务
- 4-6：刚好，有挑战
- 7-8：可能需要帮助
- 9-10：几乎不可能，除非有突破

**理想 OKR 的平均信心指数：5/10（"一半把握"）**

## OKR 和 KPI 的关系

| OKR | KPI |
|-----|-----|
| 推动改变 | 测量稳态 |
| 季度制 | 持续制 |
| 有终点 | 无终点 |
| "我们要去哪？" | "我们开得怎么样？" |
| 完成后设定新的 | 持续监控 |

**推荐的组合方式：**
- KPI 做日常健康检查（不达标=黄灯）
- OKR 做季度突破性目标（集中精力做少数重要的事）

## OKR 周期节奏

### 季度周期
```
Week 1-2: 目标提案（自下而上）+ 对齐讨论
Week 3: 全公司 OKR 发布
Week 4-11: 执行 + 每周 check-in（15 min）
Week 12: 季度回顾 + 评分 + 复盘
Week 13: 下季度 OKR 提案
```

### 每周 Check-in 格式
```
1. 本周在 OKR 上取得了什么进展？
2. 有什么阻碍？
3. 需要调整优先级吗？
```

## OKR 健康检查

- [ ] 80% 的 OKR 来自个人提案（非上级下放）？
- [ ] 每个人有 3-5 个 O，每个 O 有 2-4 个 KR？
- [ ] KR 是结果（outcome）而非任务（output）？
- [ ] 没有人的 OKR 是 100% "完成常规工作"？
- [ ] 失败也不会被惩罚的 OKR 才是真的 OKR？

## OKR 陷阱

1. **OKR = 绩效考核** — OKR 不应该直接和奖金挂钩（会导致保守设定）
2. **Org-chart OKR** — 每个人的 OKR 不应该只是"做老板的 OKR 的一部分"
3. **100% 完成率** — 如果所有人都 100% 完成 OKR，说明目标设太低了
4. **OKR 过载** — 超过 5 个 O 的季度 = 什么都做不好
