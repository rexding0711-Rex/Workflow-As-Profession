# 问题拆解（Issue Tree）

## 何时触发
用户请求解决复杂商业问题，需要结构化拆解。例如：
- "怎么帮公司降低成本30%"
- "为什么销售额下降了"
- "怎么提高客户满意度"
- 任何以"怎么/为什么/如何"开头的开放式商业问题

## 核心框架：Issue Tree（问题树）

### 原则：从大问题拆到可回答的小问题

**目标**：把模糊的大问题，拆成一组**具体的、可验证的、可回答**的子问题。

**Why拆解（为什么发生）**：
```
为什么销售额下降？
├── 外部因素
│   ├── 市场整体萎缩？（GDP/行业数据）
│   ├── 竞争加剧？（竞品份额变化）
│   └── 替代品出现？（新技术/新模式）
├── 内部因素
│   ├── 产品问题？（质量/功能/价格）
│   ├── 渠道问题？（覆盖/效率/合作关系）
│   ├── 营销问题？（品牌/推广/转化）
│   └── 服务问题？（售后/体验/口碑）
└── 客户因素
    ├── 需求变化？（偏好/场景/标准）
    ├── 购买力下降？（收入/经济）
    └── 流失率上升？（竞品挖角/不满）
```

**How拆解（怎么解决）**：
```
怎么降低成本30%？
├── 降低可变成本
│   ├── 原材料（供应商谈判/替代材料/集中采购）
│   ├── 人工成本（自动化/外包/流程优化）
│   └── 物流成本（路线优化/仓储整合/第三方物流）
├── 降低固定成本
│   ├── 租金（搬迁/共享/远程办公）
│   ├── 设备（维护优化/租赁vs购买/产能利用率）
│   └── 管理费用（数字化/裁员/组织精简）
└── 结构性降本
    ├── 业务重组（砍掉亏损线/合并部门）
    ├── 流程再造（精益/自动化/标准化）
    └── 供应链重构（垂直整合/战略联盟）
```

**Issue Tree构建原则**：
1. **从问题出发，不是从答案出发**：先问"为什么/怎么"，再想"可能是什么"
2. **MECE**：分类不重叠、不遗漏（详见MECE模块）
3. **可验证**：每个子问题必须能用数据回答（是/否/多少）
4. **优先排序**：哪个子问题对总问题影响最大？先拆这个
5. **停止条件**：拆到"这个子问题可以直接回答"为止，不要过度拆解

### 80/20法则在问题拆解中的应用

不是所有子问题都值得回答。找到**关键驱动因素（Key Driver）**：
- 哪20%的子问题解释了80%的大问题？
- 用经验/数据快速判断："如果这个问题解决了，大问题能解决多少？"
- 优先回答高影响力的子问题，忽略低影响力的

### 问题拆解陷阱

- **拆解过细**：拆到不可验证的层面（"员工是不是心情不好"）
- **从答案出发**：先有结论，再找支持（ Confirmation Bias）
- **遗漏关键分支**：只拆解容易的部分，回避困难的部分
- **静态拆解**：没有考虑时间维度（趋势、变化、外部冲击）
- **独立拆解**：没有考虑子问题之间的相互作用

## 输出模板

```markdown
# [问题] Issue Tree拆解

## 1. 问题定义
- 原始问题：[用户的模糊表述]
- 澄清后的问题：[具体、可衡量的版本]
- 问题类型：[Why/How/What]
- 约束条件：[时间/资源/政治/数据可得性]

## 2. Issue Tree（第一层拆解）
- 分支A：[子问题1]
  - 子分支A1：[子子问题]
  - 子分支A2：[子子问题]
- 分支B：[子问题2]
  - 子分支B1：[子子问题]
  - 子分支B2：[子子问题]
- 分支C：[子问题3]
  - 子分支C1：[子子问题]
  - 子分支C2：[子子问题]

## 3. MECE检查
- 不重叠检查：[分支A和B之间是否有重叠？]
- 不遗漏检查：[是否遗漏了关键分支？]
- 修改：[如有重叠/遗漏，如何修正？]

## 4. 关键驱动因素（80/20）
- 子问题影响力排序：[1. X 2. Y 3. Z]
- 高影响力子问题：[解释了80%大问题的子问题]
- 低影响力子问题：[可以暂时忽略]
- 数据可得性：[哪些子问题有数据？哪些没有？]

## 5. 假设生成
- 假设1：[对关键子问题的初步猜测]
- 假设2：[备选猜测]
- 假设3：[最保守的猜测]
- 验证路径：[用什么数据验证每个假设？]

## 6. 分析计划
- 优先分析项：[先做哪个子问题？]
- 数据需求：[需要什么数据？从哪里获得？]
- 分析方法：[用什么框架/工具？]
- 时间预算：[每个子问题花多少时间？]
```

## 经典案例

- **麦肯锡"市场份额下降"**：拆成市场增长、客户流失、价格竞争、渠道变化
- **BCG"成本优化"**：拆成固定成本、可变成本、结构性成本，每类再拆
- **贝恩"客户满意度"**：拆成产品、服务、价格、品牌、便利性
- **通用"GE业务重组"**：拆成各业务单元的ROIC，再拆成本/收入

## 使用示例

**用户输入**：
> "我们公司的销售额连续3个季度下降，帮我分析为什么"

**AI执行**：
1. 澄清问题：销售额下降幅度？行业整体情况？是否有季节性？
2. 构建Why Issue Tree：外部→内部→客户
3. 做MECE检查
4. 识别关键驱动：假设是"竞争加剧+核心客户流失"
5. 给出验证路径：竞品份额数据、客户流失率分析、价格对比
6. 输出完整的Issue Tree + 分析计划
