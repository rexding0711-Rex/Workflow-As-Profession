# 复盘方法论

## 复盘类型

| 类型 | 触发时机 | 目标 | 时长 |
|------|----------|------|------|
| 项目复盘 | 项目结束 | 总结经验、改进流程 | 2-4小时 |
| 阶段复盘 | 阶段里程碑 | 检查进度、调整方向 | 1-2小时 |
| 事件复盘 | 重大事件后 | 根因分析、改进措施 | 1-2小时 |
| 季度复盘 | 季度结束 | 目标达成、策略调整 | 2-3小时 |
| 年度复盘 | 年度结束 | 全面回顾、战略校准 | 半天-1天 |
| 日常复盘 | 每日/每周 | 习惯优化、效率提升 | 15-30分钟 |

## 复盘框架

### 四步复盘法

```
1. 回顾目标（Goal）
   - 当初目标是什么？
   - 关键里程碑是什么？
   - 成功标准是什么？

2. 评估结果（Result）
   - 实际结果是什么？
   - 与目标的差距？
   - 哪些超出预期？哪些未达预期？

3. 分析原因（Analysis）
   - 成功因素（主观/客观）
   - 失败因素（主观/客观）
   - 根因分析（5 Whys）
   - 关键假设验证

4. 总结经验（Insight）
   - 规律/方法论（可复用）
   - 改进措施（可执行）
   - 行动计划（负责人+时间）
   - 知识归档（分享）
```

### 5 Whys 根因分析

```
问题：项目延期2周
为什么延期？→ 需求变更频繁
为什么变更频繁？→ 初期需求调研不充分
为什么不充分？→ 客户参与时间不足
为什么不足？→ 未提前协调客户高层时间
为什么未协调？→ 项目启动会未邀请客户决策层

根因：项目启动阶段客户决策层参与不足
改进：启动会必须邀请客户决策层，明确需求确认机制
```

### 鱼骨图（因果分析）

```
                    问题/结果
                       |
    人        机        料        法        环        测
    |         |         |         |         |         |
  人员不足   设备故障   材料延期   流程不清   环境变化   标准不明
  技能不足   维护不当   质量波动   培训不足   政策调整   测量误差
  沟通不畅   产能不足   供应商问题  变更频繁   市场变化   检验疏漏
  激励不足   工具落后   库存不足   标准缺失   竞争加剧   监控不足
```

## 复盘质量检查

| 检查项 | 标准 | 常见问题 |
|--------|------|----------|
| 目标清晰 | 目标可衡量、可验证 | 目标模糊，无法评估 |
| 数据准确 | 基于事实，不凭记忆 | 数据缺失，靠主观印象 |
| 分析深入 | 不止于表面，找到根因 | 停留在现象，未挖根因 |
| 措施可行 | 具体、可执行、可验收 | 措施空洞，无法执行 |
| 责任明确 | 每项措施有负责人 | 措施无负责人，推诿 |
| 时间明确 | 每项措施有截止时间 | 措施无时限，拖延 |
| 经验归档 | 可复用的知识已记录 | 经验未记录，无法传承 |
| 避免指责 | 对事不对人，聚焦改进 | 变成批斗会，人身攻击 |
| 领导参与 | 关键决策者在场 | 领导缺席，决策无法拍板 |
| 闭环跟踪 | 改进措施有跟踪机制 | 复盘完就忘，无后续 |

## 复盘会议引导技巧

### 开场（5分钟）
- 明确复盘目标：总结经验、改进流程
- 设定规则：对事不对人、开放坦诚、聚焦改进
- 心理安全：不追责、不批评、鼓励暴露问题

### 回顾（15分钟）
- 数据驱动：用客观数据说话
- 时间线梳理：按时间顺序回顾
- 关键节点：聚焦决策点和转折点

### 分析（30分钟）
- 5 Whys 追问：找到根因
- 多视角：技术、管理、流程、外部
- 分类：可控因素 vs 不可控因素

### 总结（20分钟）
- 提炼规律：可复用的方法论
- 改进措施：具体、可执行
- 责任分配：每项措施有负责人

### 闭环（10分钟）
- 待办清单：明确优先级和时间
- 跟踪机制：谁来跟踪、何时检查
- 知识归档：经验分享、文档更新

## 常见复盘误区

| 误区 | 表现 | 正确做法 |
|------|------|----------|
| 变成批斗会 | 追责个人、批评指责 | 对事不对人，聚焦改进 |
| 流于形式 | 走过场、套模板 | 深入分析、找到根因 |
| 数据缺失 | 凭记忆、主观印象 | 提前准备数据、客观呈现 |
| 措施空洞 | 改进措施不具体 | 可执行、可验收、可跟踪 |
| 没有闭环 | 复盘完就结束 | 跟踪措施执行、定期回顾 |
| 忽视成功 | 只关注失败 | 同样复盘成功、提炼规律 |
| 频率不当 | 从不复盘或复盘太频繁 | 按项目/事件/周期合理安排 |
| 范围不当 | 复盘范围太大或太小 | 聚焦关键问题、适度范围 |
| 参与不足 | 关键人员缺席 | 确保决策者、执行者参与 |
| 急于下结论 | 未充分分析就总结 | 给足分析时间、充分讨论 |
