# 寻源采购（Sourcing）

## 核心能力

寻源（Sourcing）不是"货比三家"，而是"找到最优供应方案"。
本 Skill 覆盖从需求理解到供应商短名单的全流程，
不含采购执行（走 `/execute`）和合同谈判（走 `/deal` 或 `/negotiate`）。

## 工作流

### Step 1：需求定义

**采购需求说明书（SOW 框架）**：
- 采购对象：什么物料/服务？
- 规格参数：技术指标、质量标准、认证要求
- 用量预测：年用量、批次、波动性
- 交付要求：交期、地点、包装、物流
- 预算范围：目标价、可接受区间
- 特殊要求：环保、合规、ESG、保密

### Step 2：供应市场扫描

**信息源**：
- B2B 平台（1688、Alibaba、GlobalSources、ThomasNet）
- 行业协会名录
- 展会参展商名单
- 产业链图谱（用 `/graph` 查询上下游）
- 同行供应商参考（年报、招股书中的供应商披露）

**市场格局判断**：
- 卖方市场还是买方市场？（供应集中度）
- 国产替代可行性？（进口依赖度）
- 原材料成本趋势？（上游大宗商品价格）

### Step 3：供应商短名单

**初筛维度**：
| 维度 | 权重 | 评估方法 |
|------|------|---------|
| 技术能力 | 30% | 样品测试、现场审核 |
| 质量体系 | 20% | ISO认证、良率数据 |
| 价格竞争力 | 20% | 报价 vs 市场均价 |
| 交付可靠性 | 15% | 过往客户评价 |
| 财务健康 | 10% | 避免供应商倒闭风险 |
| ESG/合规 | 5% | 环保、劳工、反腐败 |

**供应商画像卡**：
```
名称：____
地点：____（物流距离：____km）
成立：____年
规模：____人 / 年营收____
主要客户：____
核心优势：____
风险点：____
综合评分：__/100
```

### Step 4：询价与评估

**RFQ（Request for Quotation）模板**：
1. 采购需求描述（附技术规格书）
2. 报价要求（单价、阶梯价、付款条件、报价有效期）
3. 资质要求（营业执照、ISO证书、过往案例）
4. 交期与包装要求
5. 报价截止时间与提交方式

**比价分析**：
- 总拥有成本（TCO）= 采购价 + 物流 + 关税 + 库存成本 + 质量成本 + 管理成本
- 不只比单价，比全生命周期成本

### Step 5：供应商准入

**现场审核清单**：
- [ ] 生产设备与产能匹配
- [ ] 质量控制流程（IQC/IPQC/OQC）
- [ ] 原材料溯源管理
- [ ] 员工技能与培训记录
- [ ] 环保与安全合规
- [ ] 信息系统与数据管理

**准入决策**：
- 入选 → 进入合格供应商名录，开始小批量试单
- 候选 → 待补充资料或改善后重新评估
- 淘汰 → 归档理由，不入名录

## 输出物清单
- 采购需求说明书（SOW）
- 供应市场分析报告
- 供应商短名单与评分矩阵
- RFQ 文件
- 报价对比分析（TCO视角）
- 供应商审核报告

## 注意事项
- 关键物料至少保持 2-3 家合格供应商（避免单一依赖）
- 低价不是唯一标准——质量风险和质量成本往往比价差大得多
- 供应商审核必须现场看，不能只信对方发的资料
- 寻源结束后交接给采购执行，不要一个人从头管到尾
