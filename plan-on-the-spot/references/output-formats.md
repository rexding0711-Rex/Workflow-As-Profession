# 输出格式规范

## Quick Plan — 口头版（30秒）

### 结构

```
目标：[1句话]
阶段：[X]个
1. [阶段1]，[产出]，[时间]；
2. [阶段2]，[产出]，[时间]；
3. [阶段3]，[产出]，[时间]。
需要您支持：[资源/决策]。
预期结果：[可衡量结果]。
```

### 示例

**融资场景：**
```
我们的目标：在Q2完成Pre-A轮融资，金额5000万，估值3亿。
计划分3个阶段：
1. 准备阶段，完成BP和财务模型，3月完成；
2. 对接阶段，接触20+投资人，拿到3+TS，4月完成；
3. 交割阶段，完成尽调和条款谈判，5月交割。
需要您支持：确认财务预测的假设，以及介绍2-3家机构。
预期结果：拿到2-3个TS，选择最优条款交割。
```

**产品场景：**
```
我们的目标：陶瓷散热平台V1.0在6月底上线。
计划分3个阶段：
1. 需求确认，完成PRD和原型，3月底完成；
2. 开发测试，完成核心功能和测试，5月底完成；
3. 灰度发布，先给5家客户试用，6月底全量。
需要您支持：确认3月底的PRD评审，以及5家试点客户名单。
预期结果：平台上线，5家客户成功试用，收集反馈迭代。
```

**执行场景：**
```
我们的目标：本周完成供应商评估的第二阶段。
计划分3步：
1. 今天完成3家供应商的资质初审；
2. 周三完成现场考察；
3. 周五完成评估报告初稿。
需要您支持：周三下午2小时参与现场考察。
预期结果：3家供应商的评估报告，推荐1家进入谈判。
```

## Quick Plan — 书面版（1页）

### 模板

```markdown
# [计划名称] — 快速计划

## 目标
[1句话，SMART]

## 关键结果（3个）
1. [KR1]：可衡量指标，截止[时间]
2. [KR2]：可衡量指标，截止[时间]
3. [KR3]：可衡量指标，截止[时间]

## 行动项（3-5个）
| 行动 | 负责人 | 截止时间 | 状态 |
|------|--------|----------|------|
| xxx | xxx | xxx | 待执行 |

## 资源需求
- 人力：xxx
- 预算：xxx
- 时间：xxx

## 关键风险
1. [风险1]：应对策略
2. [风险2]：应对策略

## 下一步
[立即行动]
```

### 示例

**融资计划：**
```markdown
# Pre-A融资计划 — 快速计划

## 目标
Q2完成Pre-A轮融资5000万，估值3亿，引入战略投资人。

## 关键结果
1. 接触20+投资人，拿到3+TS（截止4月底）
2. 完成尽调和条款谈判，签署SPA（截止5月中）
3. 完成交割，资金到账（截止5月底）

## 行动项
| 行动 | 负责人 | 截止时间 | 状态 |
|------|--------|----------|------|
| 完成BP和财务模型 | CFO | 3月15日 | 进行中 |
| 对接投资人 | CEO | 3月-4月 | 待执行 |
| 尽调准备 | 团队 | 4月 | 待执行 |
| 条款谈判 | CEO+律师 | 5月 | 待执行 |
| 交割 | CFO | 5月底 | 待执行 |

## 资源需求
- 人力：CEO 30%时间，CFO 50%时间
- 预算：中介费50万，律师费20万，尽调费用10万
- 时间：3个月

## 关键风险
1. 估值不达预期：准备2套估值方案，底线2.5亿
2. 投资人犹豫：准备3家备选，锚定1家头部

## 下一步
本周完成BP初稿，周五内部评审。
```

## 输出格式选择指南

| 场景 | 推荐格式 | 原因 |
|------|----------|------|
| 电梯偶遇投资人 | 口头版（30秒） | 时间极短，抓重点 |
| 会议即兴回答 | 口头版+书面版 | 先说后补 |
| 微信快速回复 | 书面版（1页） | 可转发，可留存 |
| 周一晨会汇报 | 书面版（1页） | 团队对齐 |
| 正式投资人会议 | 详细版（3-5页） | 需要深度 |
| 董事会汇报 | 详细版（3-5页） | 决策需要 |
| 项目启动会 | 详细版（3-5页） | 执行需要 |
| 客户提案 | 详细版（3-5页） | 专业形象 |
| 个人周计划 | 口头版或1页 | 自己看，不复杂 |
| 团队月度计划 | 1页或详细版 | 看团队规模 |

## 格式转换

Quick Plan 和 Plan in Detail 可以互相转换：

- Quick Plan → Plan in Detail：展开每个KR，增加行动项、资源、风险、沟通机制
- Plan in Detail → Quick Plan：提取目标、3个KR、关键风险，压缩到1页

转换建议：
- 被突然问到时，先生成 Quick Plan（5分钟），后续补充为 Plan in Detail
- 正式会议前，先生成 Plan in Detail，再提取 Quick Plan 作为开场
