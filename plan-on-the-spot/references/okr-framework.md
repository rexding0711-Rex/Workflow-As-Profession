# OKR 拆解框架

## OKR 基本结构

```
Objective（目标）
  ├── Key Result 1（关键结果1）
  │   ├── Action 1.1（行动项1）
  │   ├── Action 1.2（行动项2）
  │   └── Action 1.3（行动项3）
  ├── Key Result 2（关键结果2）
  │   ├── Action 2.1
  │   ├── Action 2.2
  │   └── Action 2.3
  └── Key Result 3（关键结果3）
      ├── Action 3.1
      ├── Action 3.2
      └── Action 3.3
```

## 目标（Objective）制定原则

| 原则 | 说明 | 反例 |
|------|------|------|
| 有挑战性 | 完成概率50-70%，不是100% | "完成日常工作"（太容易） |
| 方向明确 | 知道往哪走 | "做更好的产品"（太模糊） |
| 鼓舞人心 | 团队愿意为之努力 | "提高收入"（太枯燥） |
| 定性描述 | 不数字，用愿景 | "Q3收入增长20%"（这是KR） |
| 简洁 | 1句话，易记忆 | 超过2行的目标 |

**好目标示例**：
- 成为陶瓷散热领域的首选供应商
- 让产品成为客户最省心的选择
- 建立行业最可信的数据资产

**差目标示例**：
- 收入达到1000万（这是KR，不是O）
- 做好产品（太模糊）
- 完成所有任务（太容易）

## 关键结果（Key Result）制定原则

| 原则 | 说明 | 反例 |
|------|------|------|
| 可衡量 | 数字明确，验证简单 | "提升用户体验"（不可衡量） |
| 有野心 | 完成概率50-70% | "完成10个客户"（太保守） |
| 具体 | 谁、什么、何时 | "增加客户"（太模糊） |
| 结果导向 | 衡量结果，不是活动 | "发布3个功能"（是活动，不是结果） |
| 平衡 | 3-5个KR，覆盖不同维度 | 只有1个KR（太单薄） |

**好KR示例**：
- 客户满意度评分从7.5提升到8.5
- 获客成本从200元降低到150元
- 产品使用留存率从60%提升到75%
- 月收入从50万增长到100万

**差KR示例**：
- 完成3场客户访谈（活动，不是结果）
- 发布产品（太容易）
- 改进产品（不可衡量）

## 行动项（Action）制定原则

| 原则 | 说明 | 反例 |
|------|------|------|
| 可执行 | 具体动作，不是目标 | "提高客户满意度"（这是KR） |
| 有截止时间 | 明确什么时候完成 | "尽快完成"（无时间） |
| 有责任人 | 谁负责 | "团队一起完成"（无责任人） |
| 可检查 | 完成后可验证 | "多跟客户沟通"（不可检查） |

**好行动项示例**：
- 本周完成5个客户访谈（张三负责，周五前）
- 下周一前提交产品需求文档（李四负责）
- 月底前完成10家供应商初步评估（王五负责）

## OKR 分解示例

### 示例：新赛道调研项目

```
目标：成为陶瓷散热领域的权威信息源

KR1：完成100+客户/供应商访谈
  - A1.1：本月完成20家光模块客户访谈（张三）
  - A1.2：本月完成15家功率半导体客户访谈（李四）
  - A1.3：本月完成10家供应商访谈（王五）

KR2：建立行业数据库（覆盖500+公司）
  - A2.1：抓取100家竞品信息（工具）
  - A2.2：手动录入50家供应商数据（张三）
  - A2.3：建立图谱关系200+条（李四）

KR3：输出3份深度报告，被10+投资人引用
  - A3.1：完成市场报告（月底）
  - A3.2：完成技术报告（月中）
  - A3.3：分发报告给50+投资人（月初）
```

### 示例：瑞燃项目

```
目标：完成瑞燃项目合资架构设计与谈判

KR1：完成3套合资架构方案对比
  - A1.1：本周完成控股方案（控股51%）
  - A1.2：本周完成参股方案（参股30%）
  - A1.3：本周完成SPV方案（隔离风险）

KR2：完成TS条款审查，标记所有红旗条款
  - A2.1：完成经济条款审查（估值/对赌）
  - A2.2：完成控制条款审查（董事会/否决权）
  - A2.3：完成保护条款审查（清算优先/反稀释）

KR3：完成尽职调查，无重大风险
  - A3.1：FDD完成（财务）
  - A3.2：LDD完成（法务）
  - A3.3：TDD完成（技术/专利）
```

## 常见误区

| 误区 | 表现 | 修正 |
|------|------|------|
| OKR=任务清单 | 把待办事项当OKR | 区分结果（KR）和活动（Action） |
| 目标太保守 | 完成概率100% | 设定有挑战性的目标（50-70%） |
| KR不可衡量 | 用形容词而非数字 | 量化，可验证 |
| KR太多 | 10个KR | 聚焦3-5个KR |
| KR互相冲突 | 收入增长vs利润增长 | 确保KR一致性 |
| 不跟踪 | 设了OKR不检查 | 每周/每月检查进度 |
| 不调整 | 环境变了OKR不变 | 季度中期可调整 |
| 只定O不定KR | 光有目标无衡量 | 每个O必须有3-5个KR |
| 只定KR不定A | 不知道做什么 | 每个KR必须有行动项 |

## OKR 检查清单

- [ ] 目标有挑战性吗？（完成概率50-70%）
- [ ] 目标方向明确吗？
- [ ] 目标鼓舞人心吗？
- [ ] 目标简洁易记吗？
- [ ] 每个KR可衡量吗？
- [ ] 每个KR有数字吗？
- [ ] KR数量3-5个吗？
- [ ] KR覆盖不同维度吗？
- [ ] KR之间不冲突吗？
- [ ] 每个KR有行动项吗？
- [ ] 每个行动项有责任人吗？
- [ ] 每个行动项有截止时间吗？
