# 计划类型定义

## 融资计划（给投资人）

**核心关切**：回报倍数、退出路径、资金用途、团队、里程碑

**Quick Plan 输出**：
- 目标：完成[X]轮融资，金额[X]万，估值[X]万
- 关键结果：1. 投资人对接完成 2. TS签署 3. 交割完成
- 行动项：准备BP、对接投资人、尽职调查、谈判条款、交割
- 资源：时间3-6个月，预算X万（中介/法务/财务）

**Plan in Detail 输出**：
- 融资路线图：Pre-A→A→B的时间规划
- 估值锚点：可比交易、DCF、市场法
- 资金用途：研发/市场/运营/人力占比
- 投资人策略：优先接触谁、谁做锚定、谁跟投
- 条款底线：哪些可让、哪些不让

## 产品计划（给技术团队）

**核心关切**：功能优先级、技术可行性、发布节奏、质量

**Quick Plan 输出**：
- 目标：[产品名] V[X] 在[日期]上线
- 关键结果：1. 核心功能完成 2. 测试通过 3. 发布成功
- 行动项：需求评审、开发排期、测试、灰度、全量
- 资源：X个开发、X个测试、X周

**Plan in Detail 输出**：
- 功能清单：P0/P1/P2功能，MVP范围
- 技术方案：架构设计、技术选型、接口定义
- 开发排期：里程碑、依赖关系、关键路径
- 测试计划：单元测试、集成测试、UAT、性能测试
- 发布计划：灰度策略、回滚方案、监控告警

## 市场计划（给市场团队/CEO）

**核心关切**：获客成本、ROI、渠道效率、品牌

**Quick Plan 输出**：
- 目标：本季度获客[X]个，CAC控制在[X]元
- 关键结果：1. 渠道A获客X 2. 渠道B获客X 3. 品牌曝光X
- 行动项：内容投放、活动举办、KOL合作、SEO优化
- 资源：预算X万，X人团队

**Plan in Detail 输出**：
- 渠道策略：线上/线下、付费/免费、短期/长期
- 内容日历：主题、形式、发布时间、负责人
- 预算分配：各渠道预算占比、ROI目标
- 品牌策略：定位、视觉、话术、PR
- 增长实验：A/B测试清单、假设验证

## 执行计划（给团队/项目经理）

**核心关切**：任务分解、责任人、截止时间、依赖关系

**Quick Plan 输出**：
- 目标：[项目名] 在[日期]交付
- 关键结果：1. 需求完成 2. 开发完成 3. 上线完成
- 行动项：需求确认（X负责，X日）、开发（X负责，X日）、测试（X负责，X日）
- 资源：X人，X周

**Plan in Detail 输出**：
- WBS分解：任务→子任务→工作包
- 责任矩阵：RACI表
- 进度计划：Gantt图、关键路径、浮动时间
- 资源计划：人力、预算、设备、权限
- 风险计划：风险登记册、应对策略、预警机制
- 沟通计划：例会、汇报、决策、变更

## 个人计划（给自己/上级）

**核心关切**：目标聚焦、时间管理、成长路径

**Quick Plan 输出**：
- 目标：本月完成[X]
- 关键结果：1. 学习X 2. 产出X 3. 连接X
- 行动项：每天X、每周X、月末X
- 资源：时间X小时/周

**Plan in Detail 输出**：
- 年度目标：3-5个目标，与组织目标对齐
- 季度OKR：目标→KR→行动项
- 月度重点：本月必须完成的3件事
- 周计划：每周任务清单、时间块分配
- 成长计划：技能提升、人脉拓展、认知升级

## 交付计划（给客户/合作方）

**核心关切**：交付物、时间节点、验收标准、变更管理

**Quick Plan 输出**：
- 目标：[项目名] 按合同要求交付
- 关键结果：1. 交付物1完成 2. 交付物2完成 3. 验收通过
- 行动项：需求确认、设计、开发、测试、交付
- 资源：X人，X周，X万

**Plan in Detail 输出**：
- 交付物清单：名称、规格、数量、交付时间
- 里程碑计划：阶段划分、验收节点、签字确认
- 验收标准：功能/性能/质量指标
- 变更管理：变更流程、影响评估、审批机制
- 沟通机制：周报、月报、里程碑会议
- 风险管理：延期风险、质量风险、需求风险

## 混合型计划

多个目标同时推进时：

```
Q3 综合计划
├── 业务目标：获客X个，收入X万
├── 产品目标：上线X功能，修复X bug
├── 团队目标：招聘X人，培训X次
└── 个人目标：学习X技能，产出X内容
```
