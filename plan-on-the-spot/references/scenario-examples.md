# 场景示例

## 示例1：被投资人突然问"你的计划是什么"

**场景**：电梯里遇到投资人，只有30秒

**输入**：`/quick plan` 融资计划，本月

**输出**：
```
我们的目标：本月完成Pre-A轮融资的BP和投资人对接。
计划分3个阶段：
1. 准备阶段，本周完成BP和财务模型；
2. 对接阶段，下周接触5-8家机构；
3. 跟进阶段，月底拿到初步反馈。
需要您支持：如果感兴趣，能否安排下周一的详细沟通？
预期结果：拿到3+机构的初步兴趣，进入下一步尽调。
```

**后续**：投资人给了名片，约定下周一会议。周一前用 `/plan in detail` 生成详细融资计划。

## 示例6：从项目进度自动生成计划

**场景**：陶瓷散热项目已执行2周，需要更新计划

**输入**：`/plan in detail` 基于陶瓷散热项目当前进度

**系统操作**：
1. 读取 `/flow` 或 `/execute` 的进度数据
2. 当前进度：信息收集完成（100%），分析处理60%（技术路线已评估，市场规模待完成）
3. 剩余工作：决策支持、内容生成、执行推进
4. 生成详细计划，包含剩余里程碑和资源调整

**输出**：更新后的Q2计划，标注已完成的里程碑和待完成的任务

## 关键结果
1. 确认延期原因和预计恢复时间（今天）
2. 激活备选供应商（明天）
3. 调整生产计划，无客户违约（本周）

## 行动项
| 行动 | 负责人 | 截止时间 | 状态 |
|------|--------|----------|------|
| 与供应商确认延期详情 | 采购经理 | 今天14:00 | 进行中 |
| 联系备选供应商 | 采购经理 | 今天18:00 | 待执行 |

| 调整生产排期 | 生产经理 | 明天 | 待执行 |

- 加班费用：2万

## 风险
1. 备选供应商也无法及时交付：寻找第三备选
2. 客户不满：提前沟通，提供补偿方案

## 下一步
立即联系供应商确认，同时启动备选供应商对接。
