# 咨询伦理（Consulting Ethics）

## 何时触发
用户在咨询/分析过程中遇到利益冲突、数据操纵、结论预设等伦理困境。例如：
- "客户想让我得出预设结论，怎么办？"
- "分析数据不支持客户假设，要不要说？"
- "发现客户内部问题，要不要在报告中写？"
- 任何涉及"伦理/利益冲突/诚实/独立性"的场景

## 核心框架：独立/诚实/保密/利益冲突

### 咨询伦理四原则

**1. 独立性（Independence）**
- 咨询师的分析和结论必须基于客观数据和逻辑，不受客户影响
- 客户付钱，但咨询师不代表客户利益，而是代表"真相"
- 如果客户施压要求得出特定结论，咨询师必须拒绝或退出
- 独立性是咨询价值的基础，没有独立性=没有价值

**2. 诚实（Honesty）**
- 数据不支持假设 → 必须说数据不支持
- 分析有局限性 → 必须说明局限性
- 不确定的地方 → 必须说"不确定"，不能假装确定
- 负面结论 → 必须报告，不能美化或隐藏
- 诚实不是"残酷"，而是"准确"

**3. 保密（Confidentiality）**
- 客户数据、商业机密、战略信息 → 绝对保密
- 不能在公共场合讨论客户项目（即使是匿名）
- 不能将客户数据用于其他项目或个人利益
- 离职后仍然有保密义务
- 唯一的例外：法律要求披露（如法庭传票）

**4. 利益冲突（Conflict of Interest）**
- 不能同时服务竞争对手（除非双方同意+信息隔离）
- 不能推荐自己有利益关系的供应商/产品
- 不能利用客户信息为自己或他人谋利
- 不能接受客户的超过合理范围的礼品/招待
- 个人投资不能与客户业务冲突

### 常见伦理困境

**困境1：客户要求得出特定结论**
- 场景：客户说"我希望报告结论是市场还有增长空间"
- 但数据不支持 → 怎么办？
- 处理：诚实报告数据，解释为什么结论不支持，提供替代方案
- 如果客户坚持 → 退出项目或标注"免责声明"

**困境2：数据有缺陷，但客户不想花成本重做**
- 场景：样本量不足，但客户说"就这样用"
- 处理：在报告中明确标注数据局限性，结论降低置信度
- 如果客户要求删除标注 → 拒绝

**困境3：发现客户内部违规/腐败**
- 场景：分析中发现客户员工数据造假/财务违规
- 处理：报告给客户的合规/审计部门，不公开
- 如果客户高层参与 → 向监管机构报告（法律义务）

**困境4：同时服务竞争对手**
- 场景：客户A和客户B是竞争对手，都想请你做战略
- 处理：信息隔离（不同团队、不同办公室、数据不共享）或拒绝一方
- 必须向双方披露利益冲突

**困境5：推荐供应商，但咨询师有利益关系**
- 场景：客户问"推荐一个IT供应商"，你推荐的供应商是你朋友开的
- 处理：披露利益关系，让客户自己决定
- 如果收了回扣 → 严重违规，可能被起诉

### 伦理决策框架

```
1. 识别利益相关者：客户、用户、员工、社会、自己
2. 识别伦理原则冲突：诚实 vs 客户关系？保密 vs 法律义务？
3. 用公开测试：如果我的行为被公开，我会骄傲吗？
4. 用换位思考：如果我是客户/员工/用户，我希望咨询师怎么做？
5. 用底线测试：这个行为是否违反法律/行业准则/公司价值观？
6. 决策：选择最符合伦理的方案，即使短期不利
```

## 输出模板

```markdown
# [情境] 伦理分析

## 1. 情境描述
- 发生了什么：[客观描述]
- 涉及方：[客户/咨询师/第三方]
- 伦理冲突：[诚实 vs 利益 / 保密 vs 法律 / 独立 vs 压力]

## 2. 伦理原则检查
- 独立性：[是否受到客户不当影响？]
- 诚实：[数据/结论是否真实？]
- 保密：[信息是否泄露？]
- 利益冲突：[是否有利益关系？]

## 3. 利益相关者分析
- 客户：[影响/期望]
- 最终用户：[影响/期望]
- 员工：[影响/期望]
- 社会：[影响/期望]
- 咨询师自己：[影响/期望]

## 4. 伦理检验
- 公开测试：[如果公开，会骄傲吗？]
- 换位思考：[如果我是对方，会希望怎么做？]
- 底线测试：[是否违反法律/准则？]
- 遗产测试：[10年后回头看，会后悔吗？]

## 5. 可选方案
- 方案A：[描述/伦理评估/风险]
- 方案B：[描述/伦理评估/风险]
- 方案C：[描述/伦理评估/风险]

## 6. 推荐方案
- 推荐：[方案X]
- 理由：[为什么最符合伦理]
- 执行步骤：[具体怎么做]
- 风险缓解：[如果客户不满，如何应对？]

## 7. 底线
- 不可突破的原则：[列出]
- 如果必须妥协：[最小伤害方案]
- 退出机制：[如果客户坚持不道德要求，何时退出？]
```

## 经典案例

- **麦肯锡"阿片类药物报告"**：为普渡制药做咨询，报告被指控淡化成瘾风险，引发大规模诉讼和和解，损害咨询行业声誉
- **安然事件**：咨询/审计公司（安达信）在财务造假中失职，销毁文件，公司破产，行业信任崩塌
- **Wirecard丑闻**：咨询公司为Wirecard做尽职调查，未发现财务造假，事后声誉受损
- **麦府"非洲项目"**：在发展中国家项目中，被指控没有充分考虑当地社区影响，引发伦理争议

## 使用示例

**用户输入**：
> "客户想让我分析数据，证明他的新产品一定会成功，但数据其实不支持，我该怎么办？"

**AI执行**：
1. 识别伦理冲突：诚实 vs 客户关系
2. 伦理检查：独立性受威胁，诚实原则冲突
3. 利益相关者：客户（短期利益）、最终用户（被误导）、咨询师（长期声誉）
4. 可选方案：A. 诚实报告 B. 委婉表达 C. 退出项目
5. 推荐：诚实报告，标注数据局限性，提供替代方案
6. 如果客户坚持 → 退出并说明原因，保护声誉
