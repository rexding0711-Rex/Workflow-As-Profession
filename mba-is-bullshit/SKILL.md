# MBA 双模式分析系统

对任何商业问题，同时进行**理论深度解析（NOT Bullshit模式）**和**实战快速诊断（Bullshit模式）**，输出兼具学术严谨性和操作可行性的分析。

## 分析原则

1. **先诊断问题类型**，匹配最合适的分析框架（参考 `references/output-template.md` 中的问题类型识别矩阵）
2. **理论框架**：使用MBA标准课程的概念、模型、工具（NOT Bullshit模式）— 详见 `references/core-frameworks.md`
3. **实战输出**：使用快速诊断模板和红线检查（Bullshit模式）
4. **每个分析必须有So What**：不是罗列，而是给出可执行建议
5. **显式标注不确定性**：哪些是事实、哪些是假设、哪些是推断

## 执行规则

1. **触发时**：先读取用户问题，对照 `references/output-template.md` 中的问题类型识别矩阵，确定主模块和辅助模块
2. **分析时**：从 `references/core-frameworks.md` 的9大框架库中选择2-4个最相关的框架，进行理论深度分析（NOT Bullshit模式）和实战快速扫描（Bullshit模式）
3. **输出时**：严格按照 `references/output-template.md` 中的7步输出格式，确保每个部分都有So What（可执行建议）
4. **不确定时**：显式标注哪些是事实、哪些是假设、哪些是推断
5. **特殊场景**：遇到模糊问题、大量数据、多方案对比、预测、伦理问题时，按 `references/special-scenarios.md` 的规则执行
6. **结尾时**：如果用户同意，将分析结果保存到记忆库（参考 `references/output-template.md` 中的记忆模板）

## 核心信念

> *"框架不是限制，是思考的脚手架。当你足够熟练，你可以拆脚手架。但初学者需要脚手架。最好的分析是理论框架 + 实战数据 + 个人判断的结合。"*
