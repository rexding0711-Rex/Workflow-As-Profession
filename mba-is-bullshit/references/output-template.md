# 输出格式模板

所有分析必须遵循以下 **7 步结构**。

---

## 问题类型识别矩阵

先判断用户问题属于以下哪种类型，然后调用对应模块：

| 用户问题类型 | 主要调用模块 | 辅助模块 |
|-------------|------------|----------|
| "分析某行业/公司战略" | 战略管理 + 财务分析 | 宏观、营销、竞争 |
| "看这家公司财报" | 财务分析 + 会计基础 | 战略、估值、风险 |
| "设计营销/增长方案" | 营销漏斗 + 数据分析 | 战略、运营、财务 |
| "优化流程/降本增效" | 精益运营 + 运营管理 | 财务、组织、数据 |
| "团队/管理问题" | 组织设计 + 领导力 | 组织行为、激励 |
| "怎么融资/估值" | 创业融资 + 公司金融 | 财务、战略、条款 |
| "是否读MBA/职业转型" | MBA决策 + 战略 + 组织 | 网络、行业分析 |
| "分析商业案例" | 案例教学法(CRACK) + 战略 | 财务、营销、运营 |
| "出海/国际化" | 全球化 + 战略 | 宏观、营销、财务 |
| "技术/数字化/创新" | 技术管理 + 运营 | 战略、组织、数据 |
| "投资/并购分析" | 私募并购 + 公司金融 | 财务、战略、DD |
| "创业/商业模式" | 创业管理 + 战略 + 融资 | 营销、运营、财务 |
| "数据实验/指标" | 数据驱动 + 商业统计 | 营销、运营、财务 |
| "决策困难/纠结" | 领导力决策 + 经济学 | 战略、伦理、数据 |
| "伦理/合规困境" | 商业伦理 + 法律风险 | 战略、组织、决策 |
| "宏观经济/政策影响" | 宏观经济 + 经济学 | 战略、财务、行业 |

---

## 7 步标准输出格式

```markdown
# [问题/案例] 分析

## 1. 问题诊断
- 问题类型：[匹配类型]
- 核心问题：[一句话]
- 关键假设：[哪些是假设]
- 信息缺口：[还需要什么数据]

## 2. 理论分析（NOT Bullshit模式）
- 适用框架：[列出使用的框架]
- 深度分析：[用MBA课程级别的分析]
- 关键概念：[解释核心概念]

## 3. 实战诊断（Bullshit模式）
- 快速扫描：[10分钟诊断结论]
- 红线检查：[风险/问题/异常]
- 数据验证：[关键数字/比率]

## 4. 交叉验证
- 理论预测：[基于理论的预期]
- 实际数据：[与理论对比]
- 偏差解释：[为什么有偏差]

## 5. 建议与行动
- 立即执行（本周）：[1-2项]
- 短期优化（本月）：[2-3项]
- 中期建设（本季）：[1-2项]
- 长期布局（本年）：[方向]

## 6. 风险与不确定性
- 关键假设：[如果假设不成立]
- 下行风险：[最坏情况]
- 监控指标：[跟踪什么信号]
- 退出条件：[何时改变方向]

## 7. 学习要点
- 理论应用：[用了什么理论]
- 实战洞察：[从中学到什么]
- 可迁移：[应用到其他场景]
- 待深入：[需要进一步研究]
```
