# 商业伦理 Skill

## 何时触发

用户请求：
- 涉及利益冲突的决策
- 商业行为是否合规/合理
- 处理伦理困境
- 建立企业价值观或道德准则
- "这样做合不合规" / "道德和利益冲突怎么办"

## 核心框架：伦理四象限 + 利益相关者 + 决策检验

### 步骤1：伦理问题识别

不是所有商业问题都是伦理问题。伦理问题特征：
- 涉及价值冲突（如利润 vs 公平）
- 不同选择都会伤害某些利益相关者
- 没有明显的"正确答案"
- 涉及诚信、公平、责任等核心价值

**伦理问题类型**：
1. **利益冲突**：个人利益 vs 公司利益 / 客户利益
2. **信息不对称**：是否利用信息优势损害对方
3. **外部性**：商业行为对第三方的负面影响
4. **权力不对等**：对弱势方的责任
5. **文化冲突**：不同文化背景下的伦理标准差异

### 步骤2：利益相关者分析

识别所有受影响方：

```
        股东/投资人
           ↑
    员工 → 公司 → 客户/用户
           ↓
    供应商 ← 社区/社会 → 监管机构
           ↓
        环境/未来世代
```

分析每个利益相关者：
- 他们受到了什么影响？
- 他们的权力和诉求是什么？
- 如果我是他们，会希望怎么做？

**黄金规则**：己所不欲，勿施于人。换位思考到每个利益相关者。

### 步骤3：伦理四象限分析

```
          合法
           |
    合法但不道德 | 合法且道德
    （灰色地带）   | （理想状态）
    ————————————————+————————————————
    不合法且不道德 | 不合法但道德
    （红线）       | （公民不服从）
           |
          不合法
```

- **合法且道德**：放心做
- **合法但不道德**：需要判断，通常建议不做或寻求更好方案
- **不合法但道德**：特殊情况（如吹哨人），需考虑后果
- **不合法且不道德**：绝对不做

### 步骤4：伦理检验工具

**1. 公开测试（Publicity Test）**
- 如果我的行为被公开报道，我会感到自豪吗？
- 如果登上头条，我会怎么解释？

**2. 普遍性测试（Universalizability Test）**
- 如果所有人都这样做，商业环境会变好吗？
- 我是否愿意生活在这样的世界里？

**3.  reversibility测试（换位思考）**
- 如果我是被影响的一方，我接受吗？
- 如果我的家人是被影响者，我接受吗？

**4. 遗产测试（Legacy Test）**
- 10年后回头看，我会为这个决定感到骄傲吗？
- 这是我想要建立的声誉吗？

**5. 合规底线测试**
- 是否违反法律？（红线）
- 是否违反公司价值观？（黄线）
- 是否违反行业最佳实践？（灰线）

### 步骤5：伦理决策框架

**功利主义视角**：哪个选择对社会总福利贡献最大？
**义务论视角**：是否有义务或原则要求我这样做？
**美德伦理视角**：有德行的商人会怎么做？
**正义论视角**：公平吗？弱势群体得到保护了吗？

**建议**：不局限于单一视角，综合判断。但底线原则 > 功利计算。

## 伦理案例库

- **强生泰诺事件**：快速召回，公开透明，重建信任
- **福特Pinto案**：成本-收益分析导致生命被货币化
- **安然丑闻**：财务造假，系统性伦理崩溃
- **滴滴安全事件**：增长优先 vs 用户安全
- **Facebook数据门**：用户隐私 vs 商业利益
- **华为制裁**：合规 vs 商业利益 vs 国家立场
- **Patagonia**：环保优先的商业模式

## 使用示例

**用户输入**：
> "我们发现了一个竞争对手的重大安全漏洞，要不要利用这个信息？"

**AI执行**：
1. 识别伦理问题：利用信息优势 vs 公平竞争 vs 用户安全
2. 利益相关者：我方、竞争对手、共同用户、行业
3. 四象限：可能合法但不道德
4. 伦理检验：公开测试→登上头条会羞愧吗？
5. 多视角分析：功利（披露对用户更好） vs 义务（不利用他人弱点）
6. 建议：负责任披露，给对手修复时间，最终保护用户
