# People Analytics — 人力资源数据分析

用数据回答 HR 最重要的问题：谁在走？为什么走？招聘效率如何？每个人创造了多少价值？

## 何时触发

用户提及以下内容时触发：
- "帮我分析一下离职率"
- "上季度走了 5 个人，正常吗？"
- "招聘漏斗转化率怎么算？"
- "我们的人效怎么样？"
- "Analyze our attrition data"
- "What's a good hiring funnel conversion rate?"

**排除：** 此技能 NOT for：
- 财务报表分析 → 用 `core-finance`
- 业务数据分析 → 用 `data-driven`
- 薪酬市场对标 → 独立技能

## 开始之前

### 必收集项

| # | 信息 | 示例 |
|---|------|------|
| 1 | **分析目标** | "分析离职率" / "招聘效率" / "人效" |
| 2 | **数据时间范围** | "最近 12 个月" |
| 3 | **团队/公司规模** | "200 人，技术+产品+销售" |

### 数据来源

如果用户提供了数据（表格/截图），直接分析。如果用户没数据，询问：
- "你有离职数据吗？我可以帮你分析。如果没有，我可以告诉你应该收集哪些数据。"

## 工作模式

### 模式 A：离职分析
分析员工流失数据，输出流失率、离职原因分布、高风险人群画像。

### 模式 B：招聘漏斗分析
分析从投递到入职的转化率，定位瓶颈环节。

### 模式 C：人效分析
计算人均营收、人均利润、ROI of people investment。

### 模式 D：全景仪表盘
综合 A+B+C，输出一份 People Dashboard。

## 模式 A：离职分析

### 关键指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **整体流失率** | 离职人数 / 平均在职人数 × 100% | 年度基准：10-15%（科技行业） |
| **主动离职率** | 主动离职 / 平均在职人数 × 100% | 关注主动离职的"可避免"部分 |
| **新人首年流失率** | 入职 <12 月离职 / 总入职人数 × 100% | >30% 说明 onboarding 或招聘有问题 |
| **关键人才流失率** | 高绩效/高潜离职 / 该群体总数 × 100% | 这个数字应该远低于整体流失率 |

### 离职原因编码

将自由文本的离职原因编码为标准分类：

| 编码 | 类别 | 示例 |
|------|------|------|
| COMP | 薪酬 | "薪资低于市场" |
| GROWTH | 成长空间 | "没有晋升通道" |
| MANAGER | 管理问题 | "和直属上级不合" |
| CULTURE | 文化不匹配 | "加班太多" |
| PERSONAL | 个人原因 | "搬家/家庭/健康" |
| ROLE | 角色不匹配 | "工作内容和面试说的不一样" |
| LAYOFF | 被动离职 | "裁员/岗位取消" |

### 风险人群画像

从离职数据中提取高风险特征：

```
高风险信号：
- 入职 6-18 个月（"新人融入期"的离职高峰）
- 未获得晋升 > 24 个月
- 距上次调薪 > 18 个月
- 直属 manager 本身入职 < 1 年
- 部门规模在缩小
```

### 行业基准速查

| 行业 | 年化自愿流失率 | 新人首年流失率 |
|------|-------------|-------------|
| 科技/SaaS | 12-18% | 20-25% |
| 金融 | 10-15% | 15-20% |
| 零售/餐饮 | 50-70% | — |
| 制造业 | 15-25% | 20-30% |
| 咨询/专业服务 | 18-25% | 15-20% |
| 医疗 | 15-20% | 25-30% |

详见 `references/industry-benchmarks.md`

## 模式 B：招聘漏斗分析

### 漏斗阶段

```
简历筛选 → HR 初筛 → 技术面 → 终面 → Offer → 入职
```

| 阶段 | 健康转化率 | 预警信号 |
|------|----------|---------|
| 简历 → HR 初筛 | 20-30% | <10%：JD 不精准或渠道不对 |
| HR 初筛 → 技术面 | 50-70% | <40%：HR 和用人部门标准未对齐 |
| 技术面 → 终面 | 30-50% | <20%：技术标准过严或面试官培训不足 |
| 终面 → Offer | 40-60% | <30%：竞争对手更吸引人或薪酬不匹配 |
| Offer → 入职 | 70-90% | <60%：Offer 阶段跟进不够或 counter offer 多 |

### 效率指标

| 指标 | 健康范围 | 计算方式 |
|------|---------|---------|
| **Time to Fill** | 30-45 天 | 从发布 JD 到接受 Offer |
| **Time to Hire** | 15-25 天 | 从候选人进入流程到接受 Offer |
| **Cost per Hire** | 年包 × 15-25% | 总招聘成本 / 录用人数 |
| **Offer Acceptance Rate** | 70-90% | 接受 Offer 数 / 发出 Offer 数 × 100% |

## 模式 C：人效分析

### 核心人效指标

| 指标 | 公式 | 基准参考 |
|------|------|---------|
| **人均营收** | 总营收 / FTE（全职等效人数） | SaaS: $150K-250K；Top: $400K+ |
| **人均利润** | 总利润 / FTE | 因行业差异大 |
| **人力成本占比** | 总人力成本 / 总营收 × 100% | SaaS: 40-60% |
| **人才 ROI** | (营收 - 非人力成本) / 总人力成本 | >2.0 健康 |
| **Revenue per Engineer** | 产品收入 / 工程师 FTE | SaaS: $200K-500K |
| **Manager Span** | 直接汇报人数 | IC: 5-9；Manager: 3-6 |

### 人效趋势分析

如果用户提供了多个时间点的数据，做以下分析：

```
1. 边际人效变化：(本期营收 - 上期营收) / (本期人数 - 上期人数)
   如果 <0，说明新增人员未产生正向回报

2. 增长杠杆率：营收增长率 / 人数增长率
   >1.5：人效在提升
   0.8-1.2：同步增长
   <0.8：人效在下降，需要关注
```

## 主动触发（无需用户询问）

当检测到以下数据信号时，主动建议分析：

| 信号 | 触发 |
|------|------|
| 月流失率 > 2%（年化 > 24%） | "⚠️ 你提到的流失率显著高于行业基准，建议做根因分析" |
| 新人首年流失 > 40% | "⚠️ 新人流失异常高，建议检查 onboarding 和招聘匹配度" |
| 单部门季度流失 > 15% | "⚠️ [部门]的流失集中在某个时间段，可能和特定事件/manager 相关" |
| Offer 接受率 < 60% | "⚠️ 你的 Offer 接受率偏低，可能是薪酬、流程或雇主品牌问题" |

## 质量原则

1. **数据驱动但不唯数据** — 离职率低不意味着团队健康（可能是"大家都躺平了"）
2. **先比基线再谈异常** — 没有行业基准的"感觉很高"不可靠
3. **关联而非因果** — 两个指标相关不等于有因果关系
4. **切分才有洞察** — 整体流失率 15% 没意义，切到"入职 6-18 个月的后端工程师"才有
5. **保护隐私** — 分析不做低于 5 人的切分（防个人识别）

## 使用示例

**用户输入：**
> 我们 80 人 SaaS 公司，去年走了 12 个人。帮我看看正常吗？

**AI 执行过程：**
1. 计算流失率：12/80 = 15%，对照行业基准（SaaS: 12-18%），在正常范围
2. 追问：主动 vs 被动？分布在哪几个月？哪些部门？
3. 输出模式 A：整体正常但建议切分分析（新人首年流失率、技术团队 vs 销售团队）
4. 如果用户提供更多数据，生成风险人群画像
