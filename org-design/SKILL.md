# 组织设计 Skill

## 何时触发

用户请求：
- 设计或调整团队/组织架构
- 解决团队协作或管理问题
- 建立企业文化或价值观
- 设计激励机制或绩效体系
- "团队怎么搭建" / "怎么管人" / "文化怎么落地"

## 核心框架：组织六盒子 + 结构选择 + 文化塑造

### 步骤1：组织诊断（六个盒子）

韦斯伯德的六盒子模型：

```
        目的（Purpose）
           ↑
    结构（Structure）→ 关系（Relationships）
           ↓
    激励（Rewards）→ 支持（Support）
           ↓
        领导（Leadership）
```

六个维度诊断：

1. **目的（Purpose）**
   - 组织的使命/愿景/战略是否清晰？
   - 每个人是否知道自己的工作和整体目标的关系？
   - 目标是SMART的吗？

2. **结构（Structure）**
   - 组织分工是否合理？
   - 汇报关系是否清晰？
   - 是否存在职责重叠或空白？

3. **关系（Relationships）**
   - 跨部门协作是否顺畅？
   - 冲突是否被有效管理？
   - 团队氛围如何？

4. **激励（Rewards）**
   - 激励是否与目标一致？
   - 是否有非物质的认可？
   - 激励是否公平透明？

5. **支持（Support）**
   - 是否有足够的工具/资源/培训？
   - 流程是否支持而非阻碍工作？
   - 信息系统是否有效？

6. **领导（Leadership）**
   - 领导风格是否匹配组织需求？
   - 是否关注所有六个盒子？
   - 是否以身作则？

**诊断方式**：对每个盒子用 🟢 健康 / 🟡 亚健康 / 🔴 病态 评估

### 步骤2：组织结构选择

| 结构类型 | 特点 | 适用场景 | 代表公司 |
|---------|------|----------|----------|
| **职能型** | 按职能分工，专业深度 | 单一产品/稳定市场 | 传统制造业 |
| **事业部型** | 按产品/地区独立核算 | 多产品/多市场 | 华为、GE |
| **矩阵型** | 双重汇报（职能+项目） | 复杂项目/资源共享 | 波音、咨询公司 |
| **网络型** | 核心小团队+外包生态 | 轻资产/快速变化 | 耐克、苹果设计 |
| **平台型** | 中台支持+前台敏捷 | 多业务/数据共享 | 阿里、海尔 |
| **生态型** | 去中心化，自组织 | 创新驱动/创意行业 |  Valve、DAO |

**选择原则**：
- 没有完美结构，只有适配阶段的结构
- 组织结构和战略必须匹配（Chandler定律）
- 组织结构调整是创伤性手术，非必要不动

### 步骤3：团队规模与沟通复杂度

沟通通道数 = n(n-1)/2，n为团队人数

| 团队规模 | 沟通复杂度 | 管理建议 |
|---------|----------|----------|
| < 7人 | 低 | 扁平，直接沟通 |
| 7-15人 | 中 | 小组+负责人 |
| 15-50人 | 高 | 必须分层/模块化 |
| > 50人 | 极高 | 严格流程+文化粘合 |

**邓巴数**：人类稳定社交关系上限约150人。超过需要制度替代人情。

### 步骤4：文化落地（洋葱模型）

```
        表象（Artifacts）
       /    可见的行为、仪式、符号
      /
     价值观（Espoused Values）
    /    公开宣称的信念、战略
   /
  基本假设（Basic Assumptions）
       无意识的、被视为理所当然的信念
```

**文化落地四步法**：
1. **定义**：3-5个核心价值观（不是贴在墙上的口号）
2. **筛选**：招聘/晋升/淘汰时用价值观过滤（招人看价值观，开除看业绩）
3. **故事**：收集和传颂体现价值观的真实故事
4. **仪式**：设计强化文化的仪式和惯例（如亚马逊的"空椅子"代表客户）

### 步骤5：激励机制设计

激励金字塔：

```
        自我实现（意义/成长）
       /
      尊重需求（认可/地位）
     /
    社交需求（归属/团队）
   /
  安全需求（稳定/保障）
 /
生理需求（薪酬/福利）
```

**设计原则**：
- 基层重物质，中层重发展，高层重使命
- 短期激励促行动，长期激励促忠诚
- 团队激励促协作，个人激励促竞争（平衡）
- 激励要及时、明确、可预期

**股权激励常见错误**：
- 全员持股 = 没人珍惜（稀缺性原则）
- 口头承诺不兑现（信任崩塌）
- 退出机制不清（后期纠纷）
- 行权条件太宽松（无激励效果）

## 组织案例库

- **华为**：矩阵型+铁三角（客户为中心）+ 轮值CEO
- **阿里**：中台战略（平台型）+ 政委体系（文化落地）
- **Netflix**：高人才密度 + 高自由度 + 高淘汰率（自由与责任）
- **Spotify**：小队（Squad）+ 部落（Tribe）+ 分会（Chapter）+ 协会（Guild）
- **字节跳动**：Context not Control + OKR全员可见
- **海底捞**：师徒制 + 连住利益锁住管理

## 使用示例

**用户输入**：
> "我们公司从20人发展到80人了，管理很混乱，怎么调整？"

**AI执行**：
1. 先诊断：六盒子评估
2. 结构建议：从扁平到职能型+小组制
3. 文化：是否需要重新定义价值观？
4. 激励：设计层级化的薪酬和晋升体系
5. 给出分阶段实施路线图
