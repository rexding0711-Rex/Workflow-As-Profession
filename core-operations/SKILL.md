# 核心运营 Skill

## 何时触发

用户请求：
- 学习MBA运营管理课程
- 理解供应链和生产系统
- 流程优化和效率提升

## 核心框架：运营战略 + 流程设计 + 库存管理 + 供应链

### 步骤1：运营战略

**运营定位**：
- 成本领先（Cost）：低成本、高效率（沃尔玛、富士康）
- 差异化（Differentiation）：质量、灵活性、速度（Zara、苹果）
- 聚焦（Focus）：特定细分市场（Costco）

**运营绩效维度**：
- 成本（Cost）
- 质量（Quality）
- 交付速度（Speed）
- 灵活性（Flexibility）
- 可靠性（Reliability）

**权衡（Trade-offs）**：
- 不可能同时最大化所有维度
- 选择战略定位意味着接受某些维度的妥协
- 运营卓越（Operational Excellence）= 在选定维度做到世界最好

### 步骤2：流程分析

**流程类型**：
- 项目型（Project）：一次性、独特（建筑、咨询）
- 工作车间（Job Shop）：小批量、高定制（定制家具、诊所）
- 批量生产（Batch）：中等批量（面包、服装）
- 流水线（Assembly Line）：标准化、大批量（汽车、家电）
- 连续流（Continuous Flow）：高度标准化、24/7（石油、化工）
- 产品-流程矩阵（Product-Process Matrix）：从定制化到标准化

**流程绩效指标**：
- 吞吐量（Throughput）：单位时间产出
- 周期时间（Cycle Time）：生产一个单位的时间
- 瓶颈（Bottleneck）：限制产出的环节
- 利用率（Utilization）= 实际产出 / 设计产能
- 流程效率 = 增值时间 / 总时间

**排队论（Queuing Theory）**：
- 到达率 vs 服务率
- 排队长度、等待时间
- 利用率越高，等待时间增长越非线性
- 100%利用率 = 无限等待时间
- 启示：不要把资源利用率推到极限，留buffer

### 步骤3：库存管理

**库存类型**：
- 原材料库存
- 在制品（WIP）库存
- 成品库存
- 备件库存
- 在途库存

**库存成本**：
- 持有成本（Holding）：资金占用、仓储、保险、损耗（通常20-30%/年）
- 订货成本（Ordering）：下单、运输、验收
- 缺货成本（Stockout）：销售损失、客户流失、加急采购
- 总成本 = 持有成本 + 订货成本 + 缺货成本

**EOQ模型（经济订货量）**：
```
EOQ = √(2DS/H)
D = 年需求量，S = 每次订货成本，H = 单位持有成本
```
- 订货量太大 → 持有成本高
- 订货量太小 → 订货成本高
- EOQ找到最优平衡

**库存控制系统**：
- Q系统（固定订货量）：订货点触发，订固定量
- P系统（固定周期）：定期盘点，订到目标量
- ABC分类：A（高价值少品种）严格管理，C（低价值多品种）简化管理

**JIT（准时制）**：
- 零库存理想
- 需要：可靠供应商、高质量、小批量、快速换型
- 风险：供应链中断、需求波动
- 丰田生产方式的核心

### 步骤4：供应链管理

**供应链结构**：
```
供应商 → 制造商 → 分销商 → 零售商 → 客户
```

**供应链目标**：
- 响应性（Responsive）vs 效率（Efficient）
- 功能性产品（稳定需求）→ 效率型供应链
- 创新性产品（不确定需求）→ 响应型供应链

**牛鞭效应（Bullwhip Effect）**：
- 需求端的小波动 → 向上游放大
- 原因：预测修正、批量订货、价格促销、短缺博弈
- 对策：信息共享（VMI）、缩短提前期、减少批量、稳定价格

**供应链设计**：
- 推动式（Push）：基于预测生产（传统制造业）
- 拉动式（Pull）：基于订单生产（Dell、Zara）
- 推拉边界（Push-Pull Boundary）：在哪里分界

**供应链风险**：
- 中断风险：自然灾害、政治、供应商破产
- 延迟风险：运输、清关、产能不足
- 信息风险：预测错误、信息失真
- 对策：多元化供应商、安全库存、柔性产能、供应链可视化

### 步骤5：质量管理

**质量维度**：
- 设计质量：产品设计的固有质量
- 符合性质量：制造过程符合设计标准的程度
- 服务质量：交付、售后、体验

**质量管理方法**：
- **TQM（全面质量管理）**：全员参与、持续改进
- **六西格玛（Six Sigma）**：DMAIC流程，3.4ppm缺陷率
  - Define（定义）→ Measure（测量）→ Analyze（分析）→ Improve（改进）→ Control（控制）
- **ISO 9000**：质量管理体系标准
- **PDCA**：Plan-Do-Check-Act

**质量成本（Cost of Quality）**：
- 预防成本（预防问题）
- 鉴定成本（检测问题）
- 内部失败成本（内部发现的问题）
- 外部失败成本（客户发现的问题）
- 最佳策略：增加预防成本，减少外部失败成本

**统计过程控制（SPC）**：
- 控制图：监控过程是否稳定
- 规格限（Specification Limits）：客户要求
- 控制限（Control Limits）：过程能力
- 过程能力指数（Cp, Cpk）：过程能力评估

### 步骤6：项目管理

**项目管理三角**：
```
      范围（Scope）
     /           \
    /             \
时间（Time） ———— 成本（Cost）
```
- 一个角变化，至少另一个角也要变化
- 质量在中央，受三个角影响

**关键路径法（CPM）**：
- 识别所有活动及其依赖关系
- 计算每个活动的最早/最晚开始时间
- 关键路径：没有浮动时间的活动序列
- 关键路径上的延迟 = 项目延迟
- 非关键路径有浮动时间（Slack）

**PERT（计划评审技术）**：
- 三点估算：乐观时间、最可能时间、悲观时间
- 期望时间 = (乐观 + 4×最可能 + 悲观) / 6
- 方差 = [(悲观 - 乐观) / 6]^2

## 使用示例

**用户输入**：
> "我们公司在考虑采用JIT库存管理，有什么风险？"

**AI执行**：
1. 解释JIT原理和优势：降低库存成本、减少浪费、快速响应
2. 分析风险：供应链中断、需求波动、质量要求高、供应商依赖
3. 评估适用性：行业特点、供应商稳定性、需求可预测性
4. 给出实施建议：分阶段、关键供应商先行、建立安全机制
5. 对比传统库存和JIT的适用场景
