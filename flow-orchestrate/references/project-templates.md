# 项目类型模板

## 新赛道调研

```yaml
project_type: 新赛道调研
keywords: [调研, 赛道, 行业, 市场, 新领域, 机会, 进入, 可行性]
flow:
  - step: 1
    name: 信息收集
    skill: monitor
    task: 收集行业报告、新闻、竞品动态、政策信息
    output: 信息合集
    
  - step: 2
    name: 分析处理
    skill: analyze
    task: 市场规模估算、技术路线对比、客户画像分析、竞争格局梳理
    output: 分析报告
    
  - step: 3
    name: 决策支持
    skill: decide
    task: 进入策略评估、风险识别、资源需求估算、回报预测
    output: 决策简报
    
  - step: 4
    name: 内容生成
    skill: create
    task: 调研报告、决策材料、汇报PPT
    output: 正式报告
    
  - step: 5
    name: 执行推进
    skill: execute
    task: 任务分配、里程碑设定、资源协调、进度跟踪
    output: 执行计划
    
  - step: 6
    name: 复盘沉淀
    skill: review
    task: 3个月后复盘进入效果、调整策略
    output: 复盘报告
```

## 竞品分析

```yaml
project_type: 竞品分析
keywords: [竞品, 对手, 对标, 分析, 对比, 研究, 跟踪, 监控]
flow:
  - step: 1
    name: 竞品监测
    skill: monitor
    task: 监测竞品网站、产品动态、新闻发布、价格变化
    output: 竞品动态快照
    
  - step: 2
    name: 深度分析
    skill: analyze
    task: 产品功能对比、技术路线对比、市场定位对比、客户评价分析
    output: 竞品分析报告
    
  - step: 3
    name: 策略决策
    skill: decide
    task: 差异化策略、应对措施、资源投入建议
    output: 策略决策简报
    
  - step: 4
    name: 方案输出
    skill: create
    task: 竞品简报、应对方案、行动计划
    output: 方案文档
    
  - step: 5
    name: 执行跟踪
    skill: execute
    task: 分发任务、设定跟踪机制、定期回顾
    output: 执行计划
```

## 供应商评估

```yaml
project_type: 供应商评估

flow:
  - step: 1
    name: 信息收集
    skill: monitor
    task: 收集供应商公开信息、资质、财务、口碑
    output: 供应商信息合集
    
  - step: 2
    name: 资质分析
    skill: analyze
    task: 资质审查、财务分析、能力评估、风险扫描

  - step: 3
    name: 准入决策

  - step: 4
    name: 执行推进

    task: 合同谈判、 onboarding、首批交付跟踪

```

## 产品启动

```yaml
project_type: 产品启动
keywords: [产品, 启动, 上线, 发布, 上市, 推出,  rollout, launch]
flow:
  - step: 1
    name: 策略决策
    skill: decide
    task: 市场定位、定价策略、发布时机、资源确认
    output: 启动策略
    
  - step: 2
    name: 内容准备
    skill: create
    task: 产品手册、营销材料、培训资料、FAQ
    output: 内容包
    
  - step: 3
    name: 执行推进
    skill: execute
    task: 发布排期、团队协调、客户通知、渠道准备
    output: 执行计划
    
  - step: 4
    name: 复盘优化
    skill: review
    task: 发布后复盘、数据跟踪、用户反馈、持续优化
    output: 复盘报告
```

## 问题解决

```yaml
project_type: 问题解决
keywords: [问题, 故障, 异常, 解决, 修复, 处理, 事故, 危机]
flow:
  - step: 1
    name: 信息收集
    skill: monitor
    task: 收集现象、日志、影响范围、相关方信息
    output: 问题快照
    
  - step: 2
    name: 根因分析
    skill: analyze
    task: 根因定位、影响评估、解决方案对比
    output: 分析报告
    
  - step: 3
    name: 决策修复
    skill: decide
    task: 修复方案选择、风险确认、回退计划
    output: 修复决策
    
  - step: 4
    name: 执行修复
    skill: execute
    task: 修复执行、验证测试、影响消除、通知相关方
    output: 修复记录
    
  - step: 5
    name: 复盘沉淀
    skill: review
    task: 事故复盘、改进措施、知识归档、预警规则
    output: 复盘报告
```

## 流程优化

```yaml
project_type: 流程优化
keywords: [优化, 改进, 效率, 降本, 提速, 流程, 再造, 精益]
flow:
  - step: 1
    name: 现状监测
    skill: monitor
    task: 收集现有流程数据、瓶颈识别、耗时分析
    output: 现状快照
    
  - step: 2
    name: 诊断分析
    skill: analyze
    task: 流程拆解、浪费识别、根因分析、对标分析
    output: 诊断报告
    
  - step: 3
    name: 优化决策
    skill: decide
    task: 优化方案对比、ROI评估、资源需求、实施计划
    output: 优化决策
    
  - step: 4
    name: 方案输出
    skill: create
    task: 新流程文档、SOP、检查清单、培训材料
    output: 方案包
    
  - step: 5
    name: 执行落地
    skill: execute
    task: 试点实施、培训推广、变更管理、效果跟踪
    output: 执行记录
    
  - step: 6
    name: 复盘沉淀
    skill: review
    task: 效果评估、持续改进、知识归档、最佳实践
    output: 复盘报告
```

## 客户投标

```yaml
project_type: 客户投标
keywords: [投标, 方案, 竞标, 客户, 报价, 标书, 投标书, 应答]
flow:
  - step: 1
    name: 需求分析
    skill: analyze
    task: 客户需求拆解、评分标准分析、竞争对手分析、差异化策略
    output: 需求分析报告
    
  - step: 2
    name: 策略决策
    skill: decide
    task: 投标策略、定价策略、资源配置、风险识别
    output: 投标决策
    
  - step: 3
    name: 方案制作
    skill: create
    task: 技术方案、商务方案、资质材料、演示材料
    output: 投标文件
    
  - step: 4
    name: 执行投标
    skill: execute
    task: 内部审批、打印封装、提交跟踪、答疑准备
    output: 投标记录
```

## 自定义流程

```yaml
project_type: 自定义
flow:
  - step: 1
    name: [用户定义]
    skill: [用户选择]
    
  - step: 2
    name: [用户定义]
    skill: [用户选择]
    
  ...
```
