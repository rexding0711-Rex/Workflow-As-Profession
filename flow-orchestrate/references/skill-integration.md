# 技能联动表

## 1 → 2: 信息输入 → 分析处理

| 信息输出 | 分析输入 | 说明 |
|---------|---------|------|
| 网站变更记录 | 变更分析、竞争分析 | 分析竞品动作的含义 |
| 新闻摘要 | 影响评估、趋势分析 | 评估对业务的影响 |
| 数据清洗结果 | 数据探索、建模分析 | 准备好的数据直接分析 |
| 噪音过滤后的信息 | 关键信息提取、模式识别 | 过滤后的信息更精准 |
| 聊天提取的行动项 | 任务依赖分析、优先级分析 | 从信息中提取任务关系 |

## 2 → 3: 分析处理 → 决策支持

| 分析输出 | 决策输入 | 说明 |
|---------|---------|------|
| 数据洞察报告 | 决策依据、建议方案 | 数据驱动决策 |
| 文档解析结果 | 合规审查、风险评估 | 合同/政策的决策输入 |
| 技术评估报告 | 技术选型决策 | 技术决策依据 |
| 风险分析报告 | 风险处置决策 | 风险应对策略 |
| 竞争分析报告 | 战略决策 | 竞争策略选择 |

## 3 → 4: 决策支持 → 内容生成

| 决策输出 | 内容输入 | 说明 |
|---------|---------|------|
| 方案决策 | 方案文档、汇报材料 | 决策结果需要文档化 |
| 评分结果 | 决策记录、可视化 | 评分需要可视化呈现 |
| 风险评估 | 风险报告、应对方案 | 风险内容需要生成 |
| 投资回报预测 | 预测报告、敏感性分析 | 预测结果需要呈现 |
| 合同条款确认 | 合同文档、修订记录 | 条款需要文档化 |

## 4 → 5: 内容生成 → 执行推进

| 内容输出 | 执行输入 | 说明 |
|---------|---------|------|
| 调研报告 | 分发任务、讨论会议 | 报告需要执行落地 |
| 方案文档 | 任务分解、资源分配 | 方案需要执行计划 |
| 培训材料 | 培训执行、知识传递 | 材料需要培训执行 |
| 产品文案 | 发布执行、渠道推广 | 文案需要发布执行 |
| 政策简报 | 宣贯执行、合规检查 | 简报需要宣贯执行 |

## 5 → 6: 执行推进 → 复盘沉淀

| 执行输出 | 复盘输入 | 说明 |
|---------|---------|------|
| 任务完成记录 | 目标达成评估、差距分析 | 执行结果需要评估 |
| 会议纪要 | 决策跟踪、执行偏差 | 会议决策需要跟踪 |
| 项目进度报告 | 进度复盘、延期根因 | 进度需要复盘 |
| 交付物清单 | 质量评估、验收记录 | 交付物需要评估 |
| 变更记录 | 变更复盘、经验教训 | 变更需要总结 |

## 逆向触发

| 场景 | 正向流程 | 逆向触发 |
|------|---------|---------|
| 执行中发现新问题 | 1→2→3→4→5→6 | 5→1（重新收集信息） |
| 决策信息不足 | 1→2→3 | 3→1（补充信息收集） |
| 分析数据缺失 | 1→2 | 2→1（补充数据源） |
| 复盘发现新机会 | 6→5→4→3→2→1 | 6→1（启动新项目） |
| 内容需要调整 | 3→4→5 | 5→3（重新决策） |
| 执行受阻 | 4→5→6 | 5→2（分析阻塞原因） |

## 特殊联动

| 场景 | 技能组合 | 联动说明 |
|------|---------|---------|
| 合同全生命周期 | 4(create) + 3(decide) + 5(execute) | 起草→审查→执行→归档 |
| 投标项目 | 2(analyze) + 3(decide) + 4(create) + 5(execute) | 需求分析→策略决策→方案制作→投标执行 |
| 危机响应 | 1(monitor) + 2(analyze) + 3(decide) + 5(execute) + 6(review) | 监测→诊断→决策→修复→复盘 |
| 培训体系 | 2(analyze) + 4(create) + 5(execute) + 6(review) | 需求分析→材料制作→培训执行→效果评估 |
| 合规审计 | 1(monitor) + 2(analyze) + 3(decide) + 6(review) | 监测→分析→处置→合规检查 |
| 技术选型 | 1(monitor) + 2(analyze) + 3(decide) + 6(review) | 技术扫描→评估→选型→3个月后验证 |
| 市场进入 | 1(monitor) + 2(analyze) + 3(decide) + 4(create) + 5(execute) + 6(review) | 完整新赛道调研流程 |
| 产品发布 | 2(analyze) + 3(decide) + 4(create) + 5(execute) + 6(review) | 定位→策略→内容→发布→复盘 |
| 流程优化 | 2(analyze) + 3(decide) + 4(create) + 5(execute) + 6(review) | 诊断→优化→文档→实施→评估 |
| 知识管理 | 1(monitor) + 2(analyze) + 4(create) + 6(review) | 收集→整理→归档→定期回顾 |

## 数据传递规范

```
[skill-A] → [skill-B] 数据传递

必须包含：
1. 项目名称/ID
2. 当前步骤编号
3. 当前步骤输出摘要
4. 下一步建议
5. 相关上下文链接
6. 时间戳
7. 异常标记（如有）

可选包含：
- 附件列表
- 相关数据片段
- 决策依据摘要
- 待确认问题
- 风险提示

传递格式：
```yaml
transfer:
  project_id: "2024-001"
  from_step: "3-decide"
  to_step: "4-create"
  timestamp: "2024-12-15T14:30:00Z"
  summary: "决策结果：选择方案B，预算200万，6个月交付"
  key_decisions: 
    - "方案B优于方案A"
    - "预算上限200万"
    - "交付时间6个月"
  outputs:
    - type: "决策记录"
      link: "[决策记录]"
    - type: "评分矩阵"
      link: "[评分矩阵]"
  next_steps:
    - "制作项目计划书"
    - "准备资源申请"
  context:
    - "客户为A公司，预算敏感"
    - "技术团队倾向方案B"
  exceptions: []
  attachments: []
```
