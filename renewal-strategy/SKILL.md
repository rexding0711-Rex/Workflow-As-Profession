# Renewal Strategy — 续约与扩展策略

续约不是在合同到期前 30 天开始的动作，而是一个贯穿整个客户生命周期的
价值确认过程。好的续约策略让客户觉得"不续约才是疯了吧"。

## 何时触发

用户提及以下内容时触发：
- "客户快到期了，怎么谈续约？"
- "我们要涨价 20%，怎么通知客户？"
- "有哪些客户适合推 upsell？"
- "客户说太贵了，怎么办？"
- "合同谈判卡住了"

**排除：** 此技能 NOT for：
- 客户健康度评估 → 用 `customer-health-score`
- QBR 内容准备 → 用 `qbr-narrative`
- 流失挽回 → 用 `churn-prevention`

## 开始之前

| # | 信息 | 示例 |
|---|------|------|
| 1 | **客户当前合同** | "年付 30 万，3 个月后到期" |
| 2 | **使用情况** | "重度使用，核心工作流依赖我们" |
| 3 | **关系健康度** | "NPS 8，有 champion，参加过我们的 case study" |
| 4 | **续约目标** | "平价续约" / "涨价 10-15%" / "推动升级到企业版" |
| 5 | **决策人** | "原来是 CTO 决策，现在采购也介入了" |

## 续约时间线

```
合同到期前 180 天 ─── 续约准备开始
    ↓
合同到期前 150 天 ─── 健康度评估 + 扩展机会识别
    ↓
合同到期前 120 天 ─── 内部定价策略确定
    ↓
合同到期前 90 天  ─── QBR 融入续约叙事 + Champion pre-brief
    ↓
合同到期前 60 天  ─── 续约提案发出
    ↓
合同到期前 30 天  ─── 谈判 + 法律审查
    ↓
合同到期前 7 天   ─── 签署完成
```

> ⚠️ 如果现在距离到期 <90 天，跳过前面的阶段，直接从续约提案开始。

## 续约策略选择

根据客户健康度 + 使用深度，选择策略：

| 使用深 | 健康度高 | 健康度中 | 健康度低 |
|--------|---------|---------|---------|
| **深** | 🟢 涨价 + 扩展 | 🟡 锁定 + 挖掘扩展点 | 🟠 先修复关系再谈续约 |
| **中** | 🟢 平价续约 + 推深度使用 | 🟡 平价续约 + 使用提升计划 | 🟠 单月滚动，不锁长期 |
| **浅** | 🟢 锁定 + 促活 | 🟡 探讨是否 fit | 🔴 考虑不续约 |

## 场景策略

### 场景 A：涨价续约

**何时可以涨价：**
- 客户使用深度高，核心工作流依赖你的产品
- 你的产品在过去一年有明显的功能提升
- 客户 ROI 可量化（"帮你们节省了 X 小时 = ¥Y"）
- 竞争对手的价格高于或等于你的涨价后价格

**涨价通知话术框架：**
```
"[Champion Name]，

首先感谢过去一年我们的合作。我们一起达成了 [1-2 个具体成果]。

随着我们平台在过去 12 个月的持续投入和功能升级，
我们将在续约时调整价格：

- 当前价格：[X]
- 新价格：[Y]（涨幅 [Z]%）
- 生效日期：[日期]

调整的主要原因是 [一句话：产品升级 / 市场定价 / 投入增加]。

我理解任何价格变化都需要内部讨论。我可以提供：
1. 一张 ROI 总结，帮你和你的团队沟通价值
2. 和我们 VP 的一个电话，讨论任何顾虑

你怎么看？
"
```

### 场景 B：扩展销售（Upsell / Cross-sell）

**扩展信号识别：**

| 信号 | 扩展方向 |
|------|---------|
| 单个团队用 → 其他团队也想要 | 扩展席位数 |
| 用基础版 → 频繁问高级功能 | 升级到高级版 |
| 只用产品 A → 问产品 B 的功能 | Cross-sell 产品线 |
| 用户数增长 >30% | 主动讨论企业版 |
| 客户问"你们有没有 XX 功能" | 可能是扩展线索 |

**扩展提案结构：**
```markdown
1. 当前使用情况回顾（肯定价值）
2. 识别到的扩展机会
3. "如果你们用上 [扩展能力]，预计可以 [具体价值]"
4. 报价 + 实施计划
5. 风险提醒："只有你们准备好了我们才推进"
```

### 场景 C：客户说"太贵了"

**不要立即降价。** 先理解为什么贵：

| 客户说 | 真实含义 | 应对 |
|--------|---------|------|
| "太贵了" | "我没看到足够的价值" | 讲 ROI 而非价格 |
| "XX 竞品更便宜" | "我在比价" | 差异化价值，不是价格战 |
| "预算紧张" | "我想留但没钱" | 探索灵活方案（按月、缩小 scope） |
| "我们用得不多" | "产品没嵌入工作流" | 先提升使用，再谈续约 |

**谈判框架：**
```
1. 先确认价值（"过去一年我们帮你..."）
2. 理解预算压力（"我理解今年的预算情况"）
3. 探索选项（多年轻合同折扣 / 缩小 scope / 阶段性扩展）
4. 底线保护（不亏本续约，不破坏市场定价）
```

## 质量原则

1. **续约不是合同到期前 30 天的事** — 是全年持续的价值交付
2. **涨价的底气来自可量化的价值** — 如果你说不出客户 ROI，别涨价
3. **Champion 需要弹药** — 给他 ROI 数据、内部 memo、case study，帮他在内部说服别人
4. **合同条款越简单续约越快** — 如果法律审查要 8 周，你的续约策略就没用了
5. **有时候不续约是对的** — 如果客户永远无法从你的产品中获取价值，放手是尊重

## 使用示例

**用户输入：**
> 一个年付 40 万的 SaaS 客户，用了 18 个月，重度使用。现在要续约了，我们想涨 10-15%，同时也想推他们扩展到另一个产品线。

**AI 执行过程：**
1. 判断为场景 A（涨价续约）+ 场景 B（扩展销售）组合
2. 分析健康度：重度使用 + 18 个月稳定 = 🟢 健康，可以涨价
3. 输出时间线（从 90 天前开始）
4. 生成涨价通知话术 + 扩展提案
5. 准备谈判策略（涨价 15%→底价 10%，扩展产品线价格单独计算）
