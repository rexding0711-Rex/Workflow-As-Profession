# 定价谈判手册

## 谈判准备清单

### 谈判前必须知道

- [ ] 目标价、底价、理想价的数字
- [ ] 客户的当前价格和合同条款
- [ ] 客户使用数据和 ROI（至少 3 个量化指标）
- [ ] 客户的决策人和决策流程
- [ ] 客户可能提出的异议（准备 3-5 个）
- [ ] 竞争对手的价格和差异化点
- [ ] 可让步项（折扣、条款、范围）和不可让步项
- [ ] 如果谈判失败，Plan B 是什么

### BATNA（最佳替代方案）

准备谈判前，明确双方的 BATNA：

| | 我方 BATNA | 客户 BATNA |
|---|-----------|-----------|
| 最好情况 | 涨价 + 多年锁定 | 压价 + 获得更多功能 |
| 最坏情况 | 客户流失到竞品 | 切换到竞品，承担迁移成本 |
| 迁移动机 | 维护声誉、避免收入损失 | 维持业务连续性、避免迁移风险 |

> 如果客户的迁移成本 > 你的涨价金额，你的谈判地位更强。

## 异议处理速查

### "太贵了"

**不要立即降价。** 先诊断：

1. "贵是和谁比？" → 判断是比预算、比竞品、还是比预期
2. "你觉得合理的范围是？" → 了解期望差距
3. 然后讲价值：不是讲价格低，是讲 ROI 高

### "XX 竞品比你们便宜 30%"

```
回应框架：
"我理解价格是一个重要的决策因素。我想先确认——
你比较的是同等的功能吗？

[展示差异化]

如果你需要 [竞品没有的功能]，那么表面便宜 30% 可能实际上更贵。
如果你不需要这些功能，我可以帮你看看有没有更精简的方案。"
```

### "今年预算被砍了"

```
回应框架：
"我理解预算压力。让我们探索几个选项：

1. [缩小 scope] — 保持当前定价但减少席位/功能
2. [灵活付款] — 季度付款而非年付
3. [阶段性扩展] — 先续约基础版，Q2 再看扩展

你看哪个方向更可行？"
```

### "我们需要总部/采购审批"

```
早做准备：
- 在正式谈判前 4-6 周，协助 champion 准备内部 business case
- 提供 ROI 数据、case study、安全认证、合规文档
- 问 champion："除了你之外，还有谁需要点头？"
- 不要等到最后一周才发现有 5 个人需要审批
```

## 让步策略

### 让步原则

1. **永远不用价格换价格** — 客户要求降价，你不能只说"好，降 10%"。要换东西。
2. **每次让步要换对应的价值** — "如果你签多年合同，我可以给你 X% 折扣"
3. **让步幅度递减** — 第一次让 10%，第二次让 5%，第三次让 2%。传达"接近底线了"
4. **最后一张牌要小** — 最后一次让步应该是最小的（如免费加一个附加功能），表示"真的到底了"

### 可让步项菜单

| 让步 | 成本 | 换什么 |
|------|------|--------|
| 多年轻合同折扣 | 锁定期内无涨价 | 承诺续约 2-3 年 |
| 季度付款 | 现金流影响小 | 快速签单（减少内部审批阻力） |
| 免费试用高级功能 3 个月 | 边际成本低 | 3 个月后升级到高级版 |
| 免费 onboarding/培训 | 人力成本 | 签长约 |
| 案例研究/联合 PR | 几乎零成本 | 折扣或锁定 |

### 不可让步项

- 低于成本续约
- 违反已公布的定价策略（造成市场定价混乱）
- 承诺不可行的定制化需求
- 免费赠送核心功能
