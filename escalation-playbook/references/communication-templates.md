# 升级沟通模板集

## 首次响应（L1-L2）

```
主题：Re: [问题简述] — 我们已收到并开始调查

[联系人]，

感谢你反馈这个问题。我们已收到并开始调查。

当前状态：调查中
预计初步结论：[X] 小时/天内

我会在 [时间] 前更新进展。如果期间有紧急问题，请直接联系我。

[签名]
```

## 首次响应（L3-L4）

```
主题：紧急 — Re: [问题简述] — 调查已启动

[联系人]，

我们已将你反馈的问题标记为紧急事件并启动调查。

当前状态：技术团队正在排查根因
内部升级：已通知我们 VP of Engineering [姓名]
影响评估：正在进行中

接下来：
- 每 [30 分钟/1 小时] 更新一次进展
- 预计 [X] 小时内有初步结论
- 我的手机：[号码]

如果在此期间你需要联系我，随时打电话。

[签名]
```

## 进展更新（无新发现时）

```
主题：[更新 #X] — [问题简述] — [状态]

[联系人]，

进展更新：

当前状态：仍在调查中
已完成排查：[简述已排查的方向]
下一步排查：[方向]

下次更新：[时间]

[签名]
```

> 要点：哪怕没进展也要更新。沉默 > 坏消息。

## 根因确认

```
主题：[更新 #X] — 已定位根因 — [问题简述]

[联系人]，

我们已定位到问题的根因：

根因：[一句话简述技术根因]
影响范围：[受影响用户数 / 功能]
预计修复时间：[ETA]

临时 workaround：[如有]
正式修复：正在开发中，预计 [ETA] 部署

下次更新：[时间]

[签名]
```

## 修复确认

```
主题：[已解决] — [问题简述] — 修复完成

[联系人]，

问题已修复。以下是验证结果：

修复内容：[简述]
验证结果：[确认修复有效的证据]
修复时间线：[发现问题 → 定位 → 修复 → 验证]

我们对此次事件对你业务的影响表示歉意。
我们将进行事后复盘，确保此类问题不再发生。

如果你那边确认一切正常，请回复确认。

[签名]
```

## L4 高管沟通模板

```
主题：关于 [事件] 的更新 — 来自 [公司] CEO

[客户高管名]，

我是 [CEO 姓名]，[公司名] 的 CEO。

关于 [事件]，我亲自了解情况后，想让你知道：

1. 我们完全理解这件事对你们的影响
2. 我们的技术团队已经 [行动]，预计 [ETA] 解决
3. 我已经指派 [负责人] 全程跟进，每 [X] 更新进展

如果你有任何疑虑，随时打我电话：[号码]

[签名]
```
