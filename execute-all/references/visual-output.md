# 可视化输出规范

## 可视化类型

| 类型 | 说明 | 适用场景 | 调用方式 |
|------|------|----------|----------|
| **Gantt图** | 任务时间线、依赖关系、里程碑 | 项目计划、进度跟踪 | 调用 gantt-chart-builder |
| **资源负载图** | 人力分配、资源峰值、瓶颈识别 | 资源规划、负载均衡 | 内置计算+表格可视化 |
| **风险热力图** | 概率-影响矩阵、风险分布 | 风险评审、风险跟踪 | 调用 risk-heatmap |
| **进度仪表盘** | 完成率、延期数、阻塞数、预警 | 日常汇报、状态同步 | 内置计算+KPI卡片 |
| **燃尽图/燃起图** | 工作量趋势、剩余工作量 | 敏捷跟踪、迭代管理 | 内置计算+趋势图 |
| **里程碑时间轴** | 关键节点、状态、偏差 | 高层汇报、客户沟通 | 内置计算+时间轴 |

## Gantt 图（甘特图）

### 数据来源

从任务分解中提取：
```yaml
gantt_data:
  tasks:
    - id: "T-001"
      name: "任务名称"
      start: "2024-12-01"
      end: "2024-12-15"
      progress: 50
      dependencies: ["T-002"]
      assignee: "负责人"
      milestone: false
      color: "blue"  # 根据状态/优先级着色
  milestones:
    - id: "M-001"
      name: "里程碑名称"
      date: "2024-12-15"
      status: "已达成/待达成/已延期"
  phases:
    - name: "阶段1"
      start: "2024-12-01"
      end: "2024-12-31"
```

### 着色规则

| 状态 | 颜色 | 说明 |
|------|------|------|
| 正常执行 | 蓝色 | 按计划推进 |
| 已完成 | 绿色 | 已交付验收 |
| 已延期 | 橙色 | 超过截止日期 |
| 已阻塞 | 红色 | 遇到障碍无法推进 |
| 关键路径 | 深红色 | 影响总工期的任务 |
| 里程碑 | 金色 | 关键节点 |

### 调用方式

```
当 execute-all 完成任务分解后，自动调用 gantt-chart-builder：
- 输入：任务列表（含起止时间、依赖、负责人、进度）
- 输出：交互式HTML甘特图（可缩放、可拖拽、可导出）
- 关键路径：自动计算CPM，红色高亮
- 浮动时间：标注可延迟时间
```

## 资源负载图

### 数据结构

```yaml
resource_data:
  resources:
    - name: "张三"
      role: "前端工程师"
      capacity: 100  # 满负荷百分比
  allocation:
    - task: "T-001"
      resource: "张三"
      week: "2024-W50"
      load: 50  # 该周投入50%
    - task: "T-002"
      resource: "张三"
      week: "2024-W51"
      load: 80
  conflicts:
    - resource: "张三"
      week: "2024-W51"
      total_load: 130  # 超负荷30%
      tasks: ["T-002", "T-003"]
```

### 输出形式

| 形式 | 说明 | 适用场景 |
|------|------|----------|
| 资源矩阵 | 人员×时间，单元格为负载百分比 | 资源规划 |
| 负载曲线 | 折线图，展示每周总负载 vs 容量 | 瓶颈识别 |
| 冲突清单 | 超负荷时段+涉及任务+建议 | 资源调整 |
| 资源优化建议 | 重新分配方案，平衡负载 | 执行优化 |

### 计算规则

- 超载阈值：>100%为红色，80-100%为黄色，<80%为绿色
- 瓶颈识别：连续2周超载为瓶颈
- 优化建议：自动推荐任务拆分、人员借调、范围调整

## 风险热力图

### 数据结构

```yaml
risk_data:
  risks:
    - id: "R-001"
      name: "供应商交付延期"
      probability: 0.6  # 概率 0-1
      impact: 4  # 影响 1-5
      score: 2.4  # 风险评分 = probability × impact
      category: "供应链"
      owner: "采购经理"
      status: "监控中"
      mitigation: "备选供应商已激活"
```

### 矩阵可视化

```
        影响
      低  中  高  极高
概 高  中   高   极高  极高
率 中  低   中   高   极高
  低  低   低   中   高
 极低 极低 低   低   中

风险点分布：
- 极高风险（红色）：R-003, R-007
- 高风险（橙色）：R-001, R-005
- 中风险（黄色）：R-002, R-004
- 低风险（绿色）：R-006, R-008
```

### 调用方式

```
当 execute-all 执行风险评审时，调用 risk-heatmap：
- 输入：风险清单（概率、影响、类别、责任人）
- 输出：交互式HTML风险热力图（5×5矩阵）
- 自动计算：风险评分、风险等级
- 颜色标识：红/橙/黄/绿
- 可交互：点击查看风险详情、编辑概率/影响
```

## 进度仪表盘

### KPI 卡片

| 指标 | 计算方式 | 正常范围 | 预警范围 | 告警范围 |
|------|----------|----------|----------|----------|
| 整体进度 | 已完成任务数/总任务数 | >80% | 60-80% | <60% |
| 延期率 | 延期任务数/总任务数 | <10% | 10-20% | >20% |
| 阻塞率 | 阻塞任务数/总任务数 | <5% | 5-10% | >10% |
| 资源利用率 | 实际投入/计划投入 | 80-100% | 70-80% | <70%或>100% |
| 里程碑达成率 | 已达成里程碑/总里程碑 | >90% | 70-90% | <70% |
| 风险敞口 | 高风险项数/总风险数 | <20% | 20-30% | >30% |

### 输出形式

```markdown
# 进度仪表盘 [日期]

## 总体健康度
- 状态：健康/预警/告警
- 综合评分：85/100
- 趋势：↑ 改善 / ↓ 恶化 / → 持平

## KPI 卡片
| 指标 | 当前 | 目标 | 偏差 | 趋势 |
|------|------|------|------|------|
| 整体进度 | 72% | 75% | -3% | ↓ |
| 延期率 | 8% | <10% | 达标 | → |
| 阻塞率 | 3% | <5% | 达标 | → |
| 资源利用率 | 92% | 80-100% | 达标 | ↑ |
| 里程碑达成率 | 100% | >90% | 达标 | → |
| 风险敞口 | 15% | <20% | 达标 | → |

## 本周预警
- 任务 T-015 延期3天（高风险）
- 资源 张三 下周超载120%（需调整）
- 里程碑 M-003 达成风险（依赖外部审批）

## 下周重点
- 冲刺 T-015 交付（关键路径）
- 协调 T-018 资源分配
- 跟进 M-003 外部审批
```

## 燃尽图/燃起图（敏捷场景）

### 数据结构

```yaml
sprint_data:
  sprint: "Sprint 5"
  total_points: 120
  daily_burndown:
    - day: 1
      remaining: 120
      ideal: 108
    - day: 2
      remaining: 115
      ideal: 96
    - day: 3
      remaining: 105
      ideal: 84
```

### 输出形式

- 燃尽图：剩余工作量 vs 理想燃尽曲线
- 燃起图：已完成工作量累计
- 速度图：团队速率（velocity）趋势
- 累积流图：各状态任务数量变化

## 可视化输出规则

### 自动触发条件

| 场景 | 自动触发 | 输出 |
|------|----------|------|
| 任务分解完成 | Gantt图 | 项目时间线 |
| 资源分配完成 | 资源负载图 | 资源矩阵+冲突清单 |
| 风险识别完成 | 风险热力图 | 风险矩阵+应对建议 |
| 每周进度更新 | 进度仪表盘 | KPI卡片+预警清单 |
| 敏捷迭代开始 | 燃尽图 | 每日燃尽数据 |
| 里程碑临近 | 里程碑时间轴 | 关键节点状态 |
