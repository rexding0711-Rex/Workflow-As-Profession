# 执行工具箱

## 任务管理

### 任务分解模板

| 层级 | 说明 | 示例 |
|------|------|------|
| 项目 | 完整交付物 | 新赛道调研 |
| 阶段 | 里程碑 | 信息收集完成 |
| 任务 | 可执行单元 | 收集行业报告10份 |
| 子任务 | 具体动作 | 下载报告、提取关键数据 |

### 任务分配模板

```yaml
task:
  id: "T-001"
  name: "任务名称"
  owner: "负责人"
  deadline: "2024-12-31"
  priority: "高/中/低"
  dependencies: ["T-002", "T-003"]  # 前置任务
  deliverables: ["交付物1", "交付物2"]
  acceptance: "验收标准"
  status: "待执行/执行中/已完成/已阻塞"
```

## 变更管理

### 变更流程

```
1. 提出变更申请（变更描述、原因、影响）
2. 影响评估（范围、时间、成本、质量）
3. 审批决策（批准/拒绝/延期）
4. 执行变更（更新计划、通知相关方）
5. 验证变更（确认效果、记录结果）
```

### 变更记录模板

```yaml
change:
  id: "C-001"
  date: "2024-12-15"
  requester: "申请人"
  description: "变更描述"
  reason: "变更原因"
  impact:
    scope: "影响范围"
    timeline: "时间影响"
    cost: "成本影响"
    quality: "质量影响"
  decision: "批准/拒绝"
  approver: "审批人"
  status: "已执行/待执行"
```

## 风险管理（执行阶段）

| 风险类型 | 监控指标 | 预警阈值 | 响应动作 |
|----------|----------|----------|----------|
| 进度延期 | 任务完成率 | < 80% | 资源追加/范围调整 |
| 成本超支 | 预算消耗率 | > 90% | 成本控制/追加申请 |
| 质量问题 | 缺陷率 | > 5% | 质量审查/返工 |
| 人员流失 | 团队稳定性 | 离职>1人/月 | 备份计划/招聘 |
| 需求变更 | 变更频率 | > 2次/周 | 变更控制/客户沟通 |
| 外部依赖 | 依赖完成率 | < 70% | 备选方案/升级 |

## 供应商管理

### 供应商 onboarding 流程

```
1. 合同签署（法务审查、双方签字）
2. 资质审核（营业执照、资质证书、财务状况）
3. 系统对接（ERP、OA、沟通工具）
4. 培训宣贯（公司介绍、流程说明、质量标准）
5. 试单验证（小批量试产、质量确认）
6. 正式合作（批量订单、定期评估）
```

### 供应商评估周期

| 评估类型 | 频率 | 内容 | 输出 |
|----------|------|------|------|
| 日常监控 | 实时 | 订单、交付、质量 | 问题清单 |
| 月度评估 | 每月 | 交付率、合格率、响应 | 评分卡 |
| 季度复盘 | 每季度 | 综合表现、改进点 | 评估报告 |
| 年度审核 | 每年 | 现场审核、深度评估 | 审核报告 |
