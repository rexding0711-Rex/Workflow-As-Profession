# 摘要方法

## 提取式摘要（Extractive）

从原文中直接抽取关键句子，不改写。

**方法：**
- 基于位置：标题、首段、末段、每段首句
- 基于统计：TF-IDF、TextRank、关键词密度
- 基于结构：小标题、列表项、加粗文本、引用

**优点：**
- 保真度高，不扭曲原意
- 速度快，计算简单
- 适合事实性文档（财报、政策）

**缺点：**
- 连贯性差，句子间衔接生硬
- 可能遗漏隐含信息
- 不适合观点性强的文档

**适用：** 券商报告、政策文件、财务报表、技术论文

## 生成式摘要（Abstractive）

理解原文后，用自己的语言重新表述。

**方法：**
- 语义理解：提取核心概念，重新组织
- 观点提炼：识别作者立场，总结论证逻辑
- 数据重组：将分散数据整合为表格

**优点：**
- 连贯性好，可读性强
- 可以整合多处信息
- 适合观点性文档（评论、分析）

**缺点：**
- 可能引入偏差或误解
- 速度较慢
- 需要更强的理解能力

**适用：** 评论文章、分析研报、会议纪要、演讲稿

## 混合式摘要（Hybrid）

提取关键段落 + 生成连贯摘要。

**方法：**
- 先用提取式定位关键段落
- 再用生成式改写关键段落
- 最后用提取式补充数据点

**优点：**
- 兼顾保真度和可读性
- 灵活调整提取/生成比例
- 大多数场景的最优选择

**适用：** 大多数文档类型（默认方法）

## 结构式摘要（Structured）

按固定模板提取信息，不是自由摘要。

**模板：**
- 背景：为什么研究这个？
- 方法：怎么做的？
- 结果：发现了什么？
- 结论：意味着什么？
- 局限：有什么不足？
- 建议：有什么行动建议？

**适用：** 学术论文、研究报告、技术文档

## 对比式摘要（Comparative）

同时处理多篇文档，输出对比。

**方法：**
- 逐篇提取核心观点
- 识别共识（所有文档一致）
- 识别分歧（文档间矛盾）
- 识别独家信息（仅某文档提及）

**适用：** 多篇研报对比、竞品分析、政策解读

## 增量式摘要（Incremental）

基于已有摘要，增量更新。

**方法：**
- 已有旧版本摘要
- 新文档与旧摘要对比
- 只输出新增/变化的内容
- 标注哪些是旧信息，哪些是新信息

**适用：** 定期跟踪同一主题的系列报告

## 交互式摘要（Interactive）

用户指定关注重点，动态调整摘要。

**方法：**
- 用户输入关注关键词（如"陶瓷基板"、"成本"）
- 提升相关段落权重
- 生成定制化摘要
- 支持追问："关于成本的部分再详细一点"

**适用：** 深度阅读、针对性研究

## 方法选择指南

| 文档类型 | 推荐方法 | 原因 |
|----------|----------|------|
| 券商报告 | 混合式 | 数据+观点并重 |
| 技术论文 | 结构式 | 固定格式，易提取 |
| 政策文件 | 提取式 | 条款精确，不能改写 |
| 新闻/快讯 | 生成式 | 简短，需要提炼 |
| 会议纪要 | 混合式 | 观点+行动项 |
| 合同/协议 | 提取式 | 条款精确 |
| 技术文档 | 结构式 | 功能/接口/限制 |
| 财务报表 | 提取式 | 数据精确 |
| 多篇对比 | 对比式 | 识别共识和分歧 |
| 系列跟踪 | 增量式 | 避免重复 |

## 摘要质量评估

| 维度 | 标准 | 检查方法 |
|------|------|----------|
| 完整性 | 覆盖所有核心观点 | 与原文目录对比 |
| 准确性 | 不扭曲原意 | 抽样验证 |
| 简洁性 | 篇幅压缩>80% | 计算压缩比 |
| 可读性 | 语言流畅 | 人工阅读测试 |
| 相关性 | 优先用户关注内容 | 用户反馈 |
| 可追溯性 | 每条摘要可定位原文 | 引用链接 |
