# HTML 原型构建器

快速构建可交互、可本地直接运行的单文件 HTML 原型。

## 触发条件

用户提及以下任一表述时触发：
- “帮我做一个网站/网页/看板/Demo”
- “给我一个 HTML 版本”
- “前端要像苹果/Notion/暗色科技/莫兰迪/企业蓝绿”
- “纯前端 HTML+JS”、“最粗糙的 demo”
- “做之前问我问题”

## 执行流程

### 1. 咨询式提问（必须）

在动手写代码之前，先向用户确认 3-5 个关键问题，确保方向准确。参考问题：

| 维度 | 示例问题 |
|------|---------|
| 页面类型 | 这是数据看板、组织架构图、交互向导、落地页、还是其他？ |
| 视觉风格 | 偏好 Apple Health、Notion 极简、暗色科技风、莫兰迪低饱和、还是企业蓝绿？ |
| 核心功能 | 需要哪些交互（下钻、对比、切换、表单、PDF 导出）？ |
| 数据来源 | 数据是模拟的、用户提供的，还是需要接后端/API？ |
| 输出形式 | 最终是自包含 HTML 双击打开，还是需要本地服务器？ |

### 2. 设计与生成

- **单文件优先**：所有 CSS（可内联或使用 Tailwind CSS CDN）和 JS 写在一个 `.html` 文件内。
- **响应式**：确保在手机和桌面都能正常浏览。
- **图表**：如需图表，优先使用 Chart.js CDN 或原生 SVG，避免引入复杂构建工具。
- **交互**：使用原生 JS 或 Vue 3 CDN（`vue.global.js`）实现页面切换、表单、弹窗等。
- **风格**：根据用户选择应用对应配色与字体：
  - **Apple Health**：大圆角卡片、柔和阴影、圆环进度、毛玻璃效果、Inter/SF 字体。
  - **Notion/Linear**：白底黑字、极简排版、细边框、无多余装饰、灰度配色。
  - **暗色科技**：深蓝/黑底、蓝紫渐变、高亮数据、玻璃拟态、霓虹点缀。
  - **莫兰迪**：低饱和灰绿、驼色、砖红、蓝灰、暖灰白背景。
  - **企业蓝绿**：深蓝 `#0066B3` + 青绿 `#008B8B` + 蓝绿 `#1A8B9A`。

### 3. 交付与验证

- 将文件保存到当前工作目录（通常为 `C:\Users\Rex\Documents\kimi\workspace\`）。
- 向用户报告文件路径。
- 提供打开方式：
  - **直接双击**：浏览器打开 `file:///` 路径。
  - **本地服务器**：`python -m http.server 8080`（如需跨域或 Vue Router）。
- 如果用户说“做之前问我问题”，必须严格执行步骤 1，不得跳过。

### 4. 迭代修改

用户常提出以下调整，直接原地修改并重新保存：
- 换配色（如“换成莫兰迪色”）。
- 增加/删减交互功能（如“加一个对比模式”、“下钻到详细页”）。
- 调整数据或人员分配（如“直接生成四个项目组”）。
- 接入真实 AI API（如接入 DeepSeek API）——此时需要提醒用户 API Key 安全，并给出后端最小化方案（Flask + CORS）。

## 注意事项

- **不要**默认使用 Vue Router 或需要构建步骤的框架，除非用户明确要求复杂 SPA。
- **不要**在单文件 Demo 中引入需要 `npm install` 的依赖。
- 如果用户上传图片（如组织架构图），先读取图片内容，再将其转化为 HTML 结构或 CSS 布局。
- 如果用户要求“接入 AI 接口”，说明纯前端无法隐藏 Key，必须加一个极简后端（如 Flask）或使用本地代理。
- 如果用户要求“生成 PDF”，优先使用 `html2pdf.js` 或浏览器打印，避免引入重型后端库。
