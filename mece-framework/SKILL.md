# MECE（Mutually Exclusive, Collectively Exhaustive）

## 何时触发
用户需要分类、分组、拆解问题时，确保逻辑干净。例如：
- "帮我分类这些原因"
- "这个拆解有没有遗漏？"
- "怎么组织汇报结构？"
- 任何涉及分类、分组、树状拆解的场景

## 核心概念

**MECE = 不重叠（Mutually Exclusive）+ 不遗漏（Collectively Exhaustive）**

这是咨询公司最底层的逻辑洁癖。任何分类、拆解、汇报、思考都必须满足MECE。

### 不重叠（Mutually Exclusive）
- 分类之间没有交集
- 一个元素只能属于一个类别
- 避免："客户"和"大客户"重叠（大客户也是客户）
- 修正：按"规模"分（大/中/小）或按"类型"分（企业/个人/政府），不能混用维度

### 不遗漏（Collectively Exhaustive）
- 分类加起来覆盖所有可能
- 没有遗漏的灰色地带
- 避免："客户类型"只分了"企业"和"个人"（遗漏了政府/NGO）
- 修正：加一个"其他"或重新分类

### 常见MECE分类法

**按结构（Structural）**：
- 公司：总部/事业部/子公司
- 产品：产品线A/B/C
- 地理：华北/华东/华南/西部

**按流程（Process）**：
- 价值链：研发→采购→生产→营销→销售→服务
- 客户旅程：认知→考虑→购买→使用→推荐
- 时间线：过去→现在→未来

**按二分法（Dichotomy）**：
- 是/否：内部/外部、可控/不可控、线上/线下
- 问题：二分法有时不够细，但永远MECE

**按公式（Formula）**：
- 收入 = 客单价 × 客户数 × 购买频率
- 利润 = 收入 - 成本（成本再拆：固定+可变）
- 市场份额 = 我的销量 / 市场总销量

**按矩阵（2×2 Matrix）**：
- 重要/紧急矩阵：高重要高紧急、高重要低紧急、低重要高紧急、低重要低紧急
- 吸引力/竞争力矩阵（GE矩阵）
- 价格/价值矩阵（战略时钟）

**按要素（Component）**：
- 固定成本 + 可变成本 = 总成本
- 直接员工 + 间接员工 = 总员工
- 现有客户 + 新客户 = 总客户

### MECE检查清单

当你完成分类/拆解后，问自己：
1. [ ] 是否有任何两个类别重叠？（Mutually Exclusive）
2. [ ] 是否遗漏了任何可能的类别？（Collectively Exhaustive）
3. [ ] 分类维度是否一致？（不能一会儿按规模，一会儿按行业）
4. [ ] 是否过度拆解？（拆到可验证/可操作即可）
5. [ ] 是否考虑了时间维度？（静态分类可能忽略动态变化）

## 经典案例

- **麦肯锡"客户分类"**：按价值（高/中/低）× 按需求（价格敏感/服务敏感/品质敏感）= 2×3矩阵，MECE
- **BCG"成本拆分"**：固定 + 可变 = 总成本（公式法），再拆：固定=租金+人工+设备，可变=原材料+能耗+物流（要素法）
- **贝恩"市场进入"**：自建 vs 收购 vs 联盟（二分法+扩展），再评估各维度

## 使用示例

**用户输入**：
> "帮我分类我们公司客户的流失原因"

**AI执行**：
1. 提出初步分类：价格/服务/产品/竞品
2. MECE检查：发现"服务"和"售后"重叠，"竞品"和"替代品"重叠
3. 修正：按"可控/不可控"二分，再拆可控=产品/服务/价格，不可控=竞品/经济/政策
4. 输出干净的MECE分类
