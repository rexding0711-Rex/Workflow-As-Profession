# 分析方法论

## 分析流程

```
1. 明确分析目标 → 2. 数据质量检查 → 3. 选择方法 → 4. 执行分析 → 5. 验证结果 → 6. 输出报告
```

## 数据质量检查清单

- [ ] 完整性：数据缺失率 < 10%
- [ ] 一致性：字段定义统一，无矛盾
- [ ] 准确性：关键字段经抽样验证
- [ ] 时效性：数据在有效期内
- [ ] 代表性：样本覆盖目标群体
- [ ] 可访问性：数据可提取、可处理

## 分析方法选择

| 目标 | 数据类型 | 推荐方法 | 工具 |
|------|----------|----------|------|
| 了解现状 | 数值 | 描述统计 | 均值、中位数、标准差 |
| 发现趋势 | 时间序列 | 趋势分析 | 移动平均、回归 |
| 找出关联 | 多变量 | 相关性分析 | 相关系数、回归 |
| 识别分组 | 特征数据 | 聚类分析 | K-means、层次聚类 |
| 预测未来 | 历史数据 | 预测模型 | 时间序列、机器学习 |
| 评估方案 | 结构化数据 | 决策分析 | 决策树、评分卡 |
| 理解因果 | 实验数据 | 因果推断 | A/B测试、DID |
| 识别异常 | 监控数据 | 异常检测 | 统计阈值、孤立森林 |

## 分析输出规范

### 报告结构

```
1. 分析目标（1-2句）
2. 数据来源（简要说明）
3. 分析方法（说明选择理由）
4. 关键发现（3-5条，带数据支撑）
5. 数据洞察（深度解读）
6. 局限性（数据/方法限制）
7. 建议（基于分析的可操作建议）
8. 附录（详细数据、代码）
```

### 发现表述原则

- 每条发现必须有数据支撑
- 避免主观判断，用数据说话
- 区分相关性与因果性
- 标注置信度（高/中/低）
- 异常发现需要验证
- 负面发现同样需要呈现

## 分析陷阱

| 陷阱 | 表现 | 规避方法 |
|------|------|----------|
| 幸存者偏差 | 只分析成功案例 | 补充失败案例数据 |
| 确认偏差 | 只寻找支持假设的数据 | 主动寻找反面证据 |
| 相关即因果 | 把相关当因果 | 设计因果验证 |
| 样本偏差 | 样本不代表总体 | 检查样本代表性 |
| 过度拟合 | 模型过于复杂 | 简化模型、交叉验证 |
| 锚定效应 | 被初始数据影响 | 多维度交叉验证 |
| 数据陷阱 | 脏数据导致错误结论 | 数据质量检查 |
| 维度诅咒 | 变量过多 | 降维、特征选择 |

## 分析深度分级

| 级别 | 描述 | 适用场景 |
|------|------|----------|
| L1-快速扫描 | 30分钟，概览性分析 | 紧急决策、初步评估 |
| L2-标准分析 | 2-4小时，常规分析 | 日常报告、常规决策 |
| L3-深度分析 | 1-3天，深入分析 | 重大决策、战略分析 |
| L4-专项研究 | 1-4周，专项研究 | 关键项目、复杂问题 |
| L5-长期跟踪 | 持续，定期更新 | 战略监控、长期跟踪 |
