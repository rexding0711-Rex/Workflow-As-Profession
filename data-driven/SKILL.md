# 数据驱动 Skill

## 何时触发

用户请求：
- 基于数据做商业决策
- 设计数据分析或指标体系
- 进行A/B测试或实验设计
- 建立预测模型或洞察
- "这个数据怎么看" / "怎么用量化的方式决策"

## 核心框架：数据金字塔 + 实验设计 + 因果推断

### 步骤1：数据需求分层（DIKW金字塔）

```
        智慧（Wisdom）
       /    应该做什么？基于洞察的决策
      /
     知识（Knowledge）
    /    为什么发生？模式、关联、洞察
   /
  信息（Information）
 /    发生了什么？处理后的数据、指标
/
数据（Data）
     原始事实、观测记录
```

**问题诊断**：
- 用户说"数据不够" → 可能是数据采集层问题
- 用户说"数据很多但不知道怎么用" → 信息层问题
- 用户说"知道数据但不知道原因" → 知识层问题
- 用户说"知道原因但不知道怎么做" → 智慧层问题

### 步骤2：指标体系设计（OSM模型）

Objective（目标）→ Strategy（策略）→ Measurement（度量）

**北极星指标**：
- 反映产品核心价值交付
- 能驱动长期增长
- 团队能理解并影响

**分层指标**：
- **北极星指标**：1个（长期健康度）
- **一级指标**：3-5个（业务结果）
- **二级指标**：10-15个（过程驱动）
- **三级指标**：按需（运营细节）

**指标设计原则**：
- 可衡量（Measurable）
- 可操作（Actionable）
- 可归因（Attributable）
- 可比较（Comparable）

**常见指标库**：

| 类型 | 指标 | 适用场景 |
|------|------|----------|
| 获客 | CAC, CPA, 渠道转化率 | 投放优化 |
| 激活 | 激活率, 首单时间, 完成率 | 新用户体验 |
| 留存 | D1/D7/D30留存,  cohort留存 | 产品健康度 |
| 收入 | ARPU, ARPPU, LTV, MRR | 变现效率 |
| 推荐 | NPS, K因子,  referral rate | 口碑增长 |
| 效率 | 人效, 坪效, 周转率 | 运营优化 |

### 步骤3：实验设计（A/B测试规范）

**实验设计流程**：
1. **假设形成**："如果我们做X，指标Y会提升Z%"
2. **样本量计算**：用统计功效（Power）和显著性水平（α）计算最小样本量
3. **随机分组**：确保实验组和对照组随机、均衡
4. **运行实验**：达到预设样本量或时间后停止
5. **结果分析**：统计显著性 + 实际显著性
6. **决策**：上线/迭代/放弃

**实验设计原则**：
- 一次只改变一个变量（如果可能）
- 运行完整周期（避免时间偏差，如周末效应）
- 关注统计显著性和实际显著性（1%提升对亿级DAU才有意义）
- 关注长期效应（短期提升可能损害长期）

**常见错误**：
- 偷看数据（Peeking）：运行中不断看结果，导致假阳性
- 选择偏差：用户自选实验组
- 幸存者偏差：只看活跃用户
- 网络效应：一个用户的行为影响另一个用户（社交产品）

### 步骤4：因果推断（超越相关）

**核心问题**：数据告诉你"什么相关"，但商业需要知道"为什么"和"如果...会怎样"

方法：
- **随机实验**：黄金标准，但成本高
- **自然实验**：利用政策变化、自然灾害等外生冲击
- **双重差分（DID）**：对比处理组和对照组在政策前后的变化
- **工具变量（IV）**：找一个只影响自变量不影响因变量的外生变量
- **断点回归（RDD）**：利用某个阈值附近的断点
- **匹配/倾向得分**：让处理组和对照组在可观测特征上相似

**关键提醒**：相关不等于因果，因果推断是数据科学中最难的部分。

### 步骤5：数据可视化与叙事

**可视化原则**：
- 目的明确：每张图回答一个具体问题
- 选择合适图表：
  - 趋势 → 折线图
  - 构成 → 饼图/堆积图（饼图不超过5个类别）
  - 对比 → 柱状图
  - 分布 → 直方图/箱线图
  - 关系 → 散点图/热力图
- 减少认知负担：去掉网格线、3D效果、不必要的颜色

**数据叙事**：
- 从问题出发，不是从数据出发
- 用"情况-洞察-建议"结构
- 突出异常和变化，不是平铺直叙
- 给听众一个"So What"

## 输出模板

```markdown
# [业务/问题] 数据驱动分析

## 1. 问题定义
- 业务问题：[是什么]
- 数据层级：[Data/Information/Knowledge/Wisdom]
- 分析目标：[回答什么问题]

## 2. 指标设计
- 北极星指标：[是什么，为什么]
- 一级指标：[列表]
- 二级指标：[列表]
- 数据采集方式：[来源/频率]

## 3. 实验设计（如适用）
- 假设：[如果X，则Y提升Z%]
- 实验组/对照组：[设计]
- 样本量：[N=？]
- 运行周期：[时间]
- 成功标准：[统计+业务标准]

## 4. 数据分析结果
- 描述性统计：[现状]
- 对比分析：[与基准/对手对比]
- 趋势分析：[变化方向]
- 异常发现：[需要关注]

## 5. 因果推断（如适用）
- 方法：[DID/IV/匹配/实验]
- 关键发现：[X导致Y，证据是...]
- 置信度：[高/中/低]
- 局限：[还有什么可能解释]

## 6. 洞察与建议
- 核心洞察：[1-2个]
- 行动建议：[具体、可衡量]
- 预期效果：[量化预测]
- 风险与局限：[注意事项]
```

## 数据案例库

- **Netflix**：推荐算法A/B测试→个性化首页
- **Amazon**："买了X的人也买了Y" → 协同过滤
- **Airbnb**：搜索排序优化→照片质量权重
- **Facebook**：增长团队→ "7 friends in 10 days" 北极星指标
- **滴滴**：动态定价算法→供需匹配
- **今日头条**：推荐算法→内容分发

## 使用示例

**用户输入**：
> "我们产品上线了一个新功能，怎么看效果？"

**AI执行**：
1. 引导明确目标：提升留存？提升收入？提升活跃度？
2. 设计实验方案：对照组vs实验组
3. 计算所需样本量
4. 定义指标：功能使用率、留存变化、收入影响
5. 输出完整实验设计文档
