# Employee Relations — 员工关系与冲突调解

处理职场中最敏感、最复杂的人的问题。好的员工关系不是"别出事"，而是出了事有框架、有流程、有温度地解决。

> ⚠️ **重要声明：** 本技能提供管理框架和沟通话术，不构成法律建议。涉及违法行为的投诉、诉讼风险高的案件，必须咨询专业律师。

## 何时触发

用户提及以下内容时触发：
- "两个核心员工在会议上吵起来了，怎么办？"
- "有员工投诉他的 manager 不公平对待"
- "我需要做一个内部调查"
- "怎么和员工谈裁员？"
- "How to handle a harassment complaint"
- "职场冲突怎么调解？"

**排除：** 此技能 NOT for：
- 法律合规建议 → 咨询律师
- 薪酬调整方案 → 独立技能
- 绩效体系设计 → 用 `performance-review`

## 开始之前

### 必收集项

| # | 信息 | 示例 |
|---|------|------|
| 1 | **事件类型** | 冲突/投诉/纪律/裁员/骚扰 |
| 2 | **涉及方** | "2 名工程师" 或 "1 名员工投诉主管" |
| 3 | **严重程度** | 低（口角）/ 中（影响工作）/ 高（可能升级为诉讼） |
| 4 | **已尝试的解决方案** | "直属 manager 已经谈过一次，没用" |

### 隐私保护原则

- 涉及投诉时：保护投诉人身份，禁止报复行为
- 调查过程中：仅告知"需要知情"的人
- 文档记录：标记"保密"，存储于受限位置

## 工作模式

### 模式 A：冲突调解
双方可以对话 → 调解框架。适用于团队成员间的工作冲突。

### 模式 B：投诉调查
一方投诉另一方 → 正式调查流程。适用于骚扰、歧视、违规行为投诉。

### 模式 C：敏感沟通
单方通知 → 话术设计。适用于裁员沟通、绩效警告、纪律处分通知。

## 模式 A：冲突调解框架

### 五步调解法

```
1. 单独倾听（各 30 min）
   - "帮我理解发生了什么"
   - "这件事对你最大的影响是什么？"
   - "你理想的结果是什么？"
   原则：不评判、不打断、不承诺

2. 归因分析
   - 事实分歧（对事实有不同记忆/理解）
   - 利益冲突（争夺同一资源/机会）
   - 风格差异（沟通方式不合）
   - 结构性问题（职责不清、汇报线混乱）

3. 三方会谈（如有需要）
   规则设定：
   - 每人轮流发言，不打断
   - 用"I"句式而非"You"句式（"我感到..."而非"你总是..."）
   - 聚焦解决方案而非追究责任
   - 调解人的角色：引导讨论，不做裁判

4. 共同方案
   - 双方各提 2-3 个可行的改进措施
   - 找出重叠或可合并的方案
   - 书面记录约定（双方签字确认）

5. 跟进
   - 第 1 周：30 分钟 check-in
   - 第 1 个月：正式回顾
   - 如果复发 → 升级到正式纪律流程
```

### 调解话术速查

| 场景 | 话术 |
|------|------|
| 开场 | "我们今天坐在一起，是因为我相信你们可以找到比现在更好的合作方式。" |
| 打断指责 | "我听到你的 frustration。我们先不聚焦在'谁对谁错'，而是'我们需要什么不同'。" |
| 僵局 | "如果什么都不改变，3 个月后会怎么样？那是你们想要的吗？" |
| 达成共识 | "那我总结一下你们今天达成的约定[具体列出]，请确认这是你们同意的。" |

## 模式 B：投诉调查流程

### 调查步骤

```
1. 接收投诉
   - 记录投诉时间、内容、涉及方
   - 告知投诉人：保密原则和可能的调查流程
   - 评估紧急程度（如涉及人身安全 → 立即行动）

2. 调查启动
   - 指定调查人（必须是中立方，非任何一方的汇报线）
   - 确定调查范围
   - 制定访谈名单

3. 证据收集
   - 文档证据（邮件、聊天记录、代码提交）
   - 物证（监控、门禁记录）
   - 证人访谈（每次访谈做书面记录）

4. 访谈技巧
   - 开放式开场："请告诉我你了解的情况"
   - 事实挖掘："具体是哪一天？几点？谁在场？"
   - 交叉验证："A 说的情况是这样，你怎么看？"
   - 不做评判："感谢你分享这些信息"
   - 禁止诱导性提问："你觉得他是不是故意的？" ❌

5. 调查报告
   - 事实陈述（不含主观判断）
   - 证据摘要
   - 政策对照（违反了哪条规定）
   - 结论（成立/不成立/无法确认）
   - 建议措施

6. 反馈闭环
   - 向投诉人反馈调查结果（不透露具体处分细节）
   - 向被投诉人传达决定
   - 如有必要，向团队做有限度通报
```

## 模式 C：敏感沟通话术

### 裁员沟通指南

参见 `references/difficult-conversations.md`

核心原则：
- **直接但不冷酷**：第一句话就说明决定，不要绕弯子
- **尊重**：肯定对方的贡献
- **准备**：提前准备好所有实际问题的答案（离职日期、补偿方案、交接安排）
- **简短**：5-8 分钟说完，不要展开讨论
- **线下**：必须当面或视频，禁止邮件/短信通知

**话术框架：**
```
"[姓名]，很遗憾我今天要和你说一个艰难的决定。
经过慎重考虑，公司决定 [裁员/岗位取消/业务调整]。
这不是对你个人表现的评判。我们感谢你在 [具体贡献] 上所做的工作。

接下来关于过渡安排：[离职日期 / 补偿方案 / 交接计划 / 离职支持]

你现在有什么问题吗？我会解答所有实际问题。"
```

### 绩效警告话术

```
1. 用具体案例说明差距（不是"你态度不好"，而是"在上周的客户会议上，你在客户提问时打断三次"）
2. 明确期望（"接下来 30 天，我希望看到 [具体行为改变]"）
3. 提供支持（"我可以帮你 [具体帮助]"）
4. 后续节点（"两周后我们再 check-in"）
```

## 投诉升级路径

根据严重程度和投诉对象，决定升级路径：

```
Level 1: 直接 Manager → 解决日常工作冲突
Level 2: HR Business Partner → 涉及绩效/行为的正式投诉
Level 3: HR Director / 法务 → 涉及骚扰、歧视、合规风险
Level 4: CEO / 董事会 → 涉及高管、系统性风险
```

## 主动触发（无需用户询问）

当检测到以下信号时，主动提醒：
- 用户描述的场景涉及"骚扰"或"歧视" → 提醒法律报告义务
- 用户描述的冲突涉及"报复" → 建议立即升级到法务
- 用户考虑单方面解雇 → 提醒程序正当性（中国劳动法需提前30天或支付代通知金）

## 质量原则

1. **所有调查文档假设未来会在法庭上被阅读** — 只写事实，不写揣测
2. **调解不是和稀泥** — 目标是找到可行方案，不是让双方"握手言和"
3. **保护弱势方** — 如果双方权力不对等（manager vs IC），调解可能不合适，考虑正式调查
4. **及时性很重要** — 投诉后 48 小时内启动调查，冲突持续 >2 周不介入 = 默许
5. **隐私是单向门** — 泄漏了就无法收回

## 使用示例

**用户输入：**
> 我团队有两个高级工程师，因为技术方案分歧在会上吵了 3 次了，现在其他人都很尴尬。怎么办？

**AI 执行过程：**
1. 判断为模式 A：冲突调解（双方可以对话，无权力不对等，无投诉性质）
2. 追问严重程度和已尝试方案
3. 输出五步调解方案：先单独听 → 归因（方案分歧 + 风格差异）→ 三方会谈规则 → 方案建议 → 跟进计划
4. 附调解话术速查表

## 与其他技能的关系

- 绩效相关问题 → 先看 `performance-review` 是否已有 PIP
- 组织架构调整导致的冲突 → 可参考 `org-design`
- 涉及法律风险时 → 咨询专业法律意见，不依赖 AI
