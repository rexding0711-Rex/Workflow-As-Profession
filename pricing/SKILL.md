# 定价策略（Pricing）

## 核心能力

定价不是"算成本加毛利"，而是"在客户愿意付、竞品挡不住、自己有钱赚三者中找最优解"。
本 Skill 覆盖定价方法论和价格分析全流程。

## 工作流

### Step 1：定价基准分析

**三维对标**：
| 维度 | 方法 | 数据来源 |
|------|------|---------|
| 成本底线 | 完全成本法、边际成本法 | 内部财务数据 |
| 竞品参照 | 竞品价格矩阵、功能对标 | 公开报价/G2/行业报告 |
| 客户意愿 | Van Westendorp、联合分析 | 客户访谈/问卷 |

**Van Westendorp 价格敏感度模型（四个问题）**：
1. 什么价格你觉得"太便宜了，质量可能有问题"？
2. 什么价格你觉得"划算，值得买"？
3. 什么价格你觉得"有点贵，但可以考虑"？
4. 什么价格你觉得"太贵了，不会买"？
→ 最优价格点 = 问题2和问题3的交叉区域

### Step 2：定价方法选择

| 方法 | 逻辑 | 适用场景 | 风险 |
|------|------|---------|------|
| 成本加成 | 成本 × (1 + 利润率) | 制造/代工/大宗商品 | 忽视客户价值，定价过低 |
| 价值定价 | 基于客户获得的经济价值 | SaaS/专业服务/高端产品 | 客户价值难量化 |
| 竞争参照 | 跟随/低于/高于竞品 | 标准化/同质化产品 | 陷入价格战 |
| 动态定价 | 供需实时调整 | 酒店/机票/共享出行 | 客户反感不可预测 |
| 捆绑定价 | 组合优惠 > 单独购买 | SaaS套餐/快餐/套餐 | "我不需要全部" |
| 分层定价 | 按功能/用量分级 | SaaS/会员制/流媒体 | 层级划分逻辑需合理 |
| 渗透定价 | 低价入市抢份额 | 新市场/网络效应强 | 以后再涨价很难 |

### Step 3：定价结构设计

**B2B 定价**：
- 阶梯价（Volume-based）：100件/1000件/10000件 三档
- 打包价（Bundle）：产品 + 安装 + 培训 + 维保
- 订阅价（Subscription）：按月/年，按用户数/用量

**折扣体系**：
| 折扣类型 | 幅度 | 目的 |
|---------|------|------|
| 数量折扣 | 5-20% | 激励大单 |
| 预付折扣 | 3-5% | 改善现金流 |
| 忠诚折扣 | 2-5%/年递增 | 提高留存 |
| 战略折扣 | 10-30% | 标杆客户/打入新市场 |

**价格锚定**：
- 同时展示三个选项（基础/标准/旗舰），大部分人选中间
- "原价999/现价399"——原价是锚
- 竞品价格先报——让客户以竞品价心理锚定

### Step 4：价格执行与监控

**价格 waterfalls**（实际成交价 vs 标价的差距分析）：
```
标价：100
  - 合同折扣：-15
  - 数量折扣：-5
  - 促销折扣：-8
  - 付款条件折让：-2
实际成交价：70
```
→ 折扣率 30%。健康值因行业而异，>50% 需要审视。

**价格弹性监控**：
- 涨价 x% → 客户流失率变化 y%
- 降价 x% → 增量客户数变化 z%
- 计算最优价格点：利润最大化的那个点

## 常见场景

| 场景 | 推荐方法 | 产出 |
|------|---------|------|
| 新产品上市定价 | 价值定价 + 竞品参照 | 定价方案、锚定策略 |
| 涨价决策 | 价格弹性分析 + 客户分层 | 涨价方案、过渡计划 |
| 大客户报价 | 成本分析 + 阶梯报价 | 报价单、谈判价格带 |
| 套餐/捆绑设计 | 捆绑定价 + 分层定价 | 套餐结构、定价梯度 |
| 价格战应对 | 竞品矩阵 + 价值差异化 | 竞争策略、非价格应对 |

## 注意事项
- 不要以成本为唯一依据——客户不关心你的成本，只关心自己获得的价值
- 频繁打折比高价更伤品牌——降价容易涨价难
- 定价 = 定位。高价暗示高品质，低价暗示低品质
- 国际定价考虑购买力平价（PPP）和关税
