# 故事线（Storyline）与金字塔原理（Pyramid Principle）

## 何时触发
用户需要汇报、提案、写作、演示时，让听众30秒听懂。例如：
- "帮我准备向CEO的汇报"
- "怎么写Executive Summary"
- "这个分析怎么呈现给客户"
- 任何需要"把复杂事情说清楚"的场景

## 核心框架：金字塔原理

### 芭芭拉·明托（Barbara Minto）的金字塔结构

```
          核心结论（Governing Thought）
                 /        |        \
                /         |         \
         支持论点1     支持论点2    支持论点3
          / | \         / | \        / | \
        数据/证据    数据/证据     数据/证据
```

**核心原则**：
1. **结论先行**：第一句话就说答案，不是最后才说
2. **以上统下**：上层是下层的概括，下层是上层的支持
3. **归类分组**：同类信息放在一起，用MECE分组
4. **逻辑递进**：按时间、结构、重要性排序

### 故事线（Storyline）六要素

一个好的故事线回答6个问题：
1. **Situation（背景）**：发生了什么？听众已经知道什么？
2. **Complication（冲突）**：有什么问题？为什么现在需要行动？
3. **Question（问题）**：我们需要回答什么问题？
4. **Answer（答案）**：核心结论是什么？（Governing Thought）
5. **Support（支持）**：为什么这个结论是对的？（3-5个论点）
6. **Action（行动）**：接下来做什么？（具体、可执行）

**缩写：SCQA**（Situation-Complication-Question-Answer）

### 电梯演讲（Elevator Pitch）

30秒版本：
```
我们发现[问题/机会]，
因为[关键原因]，
导致[影响/损失]，
我们建议[方案]，
预期[结果]，
需要[资源/决策]。
```

### 高管汇报结构（10分钟版）

```
第1页：Executive Summary（1页，核心结论+3个支持点+行动）
第2-4页：Situation & Complication（背景+问题，各1页）
第5-7页：Analysis（分析，每页一个论点+数据）
第8页：Recommendation（建议，具体方案）
第9页：Action Plan（行动计划，时间表+负责人）
第10页：Risk & Next Steps（风险+下一步）
```

### 金字塔结构检查清单

1. [ ] 核心结论是否在第一句/第一页？
2. [ ] 每个上层论点是否被下层充分支持？
3. [ ] 同层论点是否MECE？
4. [ ] 数据是否直接支持论点？（不是凑数）
5. [ ] 是否避免了"信息堆砌"？（只放支持结论的信息）
6. [ ] 行动建议是否具体、可执行、有负责人？

## 输出模板

```markdown
# [主题] 故事线设计

## 1. SCQA框架
- Situation（背景）：[听众已经知道的事实]
- Complication（冲突）：[问题/机会/变化]
- Question（问题）：[需要回答的核心问题]
- Answer（答案）：[核心结论，一句话]

## 2. 金字塔结构
- Governing Thought（核心结论）：[最上面一句话]
  - 支持论点1：[概括]
    - 证据1a：[数据/事实]
    - 证据1b：[数据/事实]
  - 支持论点2：[概括]
    - 证据2a：[数据/事实]
    - 证据2b：[数据/事实]
  - 支持论点3：[概括]
    - 证据3a：[数据/事实]
    - 证据3b：[数据/事实]

## 3. Executive Summary（1页）
- 核心结论：[一句话]
- 关键发现：[3点，每点一句话]
- 建议行动：[1-2项，具体可执行]
- 预期影响：[量化结果]

## 4. 电梯演讲（30秒）
[填空：我们发现...因为...导致...建议...预期...需要...]

## 5. 汇报结构（10分钟版）
- 第1页：Executive Summary
- 第2-3页：Situation & Complication
- 第4-6页：Analysis（每页一个论点）
- 第7页：Recommendation
- 第8页：Action Plan
- 第9页：Risk & Next Steps

## 6. 视觉呈现建议
- 图表类型：[趋势用折线、构成用饼图/瀑布、对比用柱状、关系用矩阵]
- 每页标题：[结论式标题，不是描述式标题]
- 数据标注：[关键数字标红/加粗，突出洞察]
- 留白：[不要堆满，每页一个核心信息]
```

## 经典案例

- **麦肯锡"市场份额下降"汇报**：第1页"建议降价10%以挽回5%份额"→ 支持：竞品降价20%/我们价格溢价15%/客户价格敏感度高
- **BCG"成本优化"汇报**：第1页"通过供应链重构降低15%成本"→ 支持：供应商集中度低/物流路线冗余/库存周转天数高于行业50%
- **贝恩"客户分层"汇报**：第1页"聚焦Top 20%客户可提升30%利润"→ 支持：Top 20贡献80%利润/长尾客户亏损/服务成本差异

## 使用示例

**用户输入**：
> "帮我准备向CEO的汇报，我们分析了一个新产品上市失败的原因"

**AI执行**：
1. 设计SCQA：背景（市场增长）→ 冲突（上市3个月销量仅为目标10%）→ 问题（为什么失败？）→ 答案（定位错误+渠道错配+定价过高）
2. 构建金字塔：核心结论→3个论点→每个论点2个证据
3. 写Executive Summary（1页）
4. 设计电梯演讲（30秒）
5. 设计10页PPT结构
