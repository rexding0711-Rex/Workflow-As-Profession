# User Interview Guide — 用户访谈指南

"和用户聊聊"是最被低估的研究方法——也是最容易被做烂的。
差的访谈：问用户"你想要什么功能"，用户说"我想要一个导出按钮"，你做了，没人用。
好的访谈：问用户"你上一次导出数据是什么时候？为什么要导出？导出后做了什么？"
然后你发现用户真正需要的不是导出按钮，是自动周报。

## 何时触发

用户提及以下内容时触发：
- "帮我设计一个用户访谈提纲"
- "我想找 5 个用户聊聊，问什么好？"
- "用户访谈怎么做才能不引导用户回答？"
- "访谈完了，录音一大堆，怎么分析？"
- "User interview discussion guide"
- "怎么从访谈中提取洞察？"

**排除：** 此技能 NOT for：
- 可用性测试 → 用 `usability-testing`
- 用户画像 → 用 `persona-builder`
- 问卷调研 → 属于定量研究，不在此范围

## 开始之前

| # | 信息 | 示例 |
|---|------|------|
| 1 | **研究目标** | "理解用户为什么在结算页流失" |
| 2 | **目标用户** | "过去 30 天加购但未支付的用户" |
| 3 | **已有假设** | "可能是运费太贵，或者支付方式不够" |
| 4 | **资源约束** | "预算有限，能访谈 8 个人" |

## 访谈提纲设计

### 漏斗式问题结构（45 分钟）

```
开场（5 min）— 建立信任
  "我们今天不是来卖东西的，是想了解你的真实体验。
   没有对错，你的任何反馈对我们都很有价值。"
   
热身（10 min）— 了解上下文
  "先聊聊你的日常吧。你主要负责什么？最近在忙什么项目？"

核心探索（20 min）— 深挖目标行为
  "上次你在我们网站下单是什么时候？能不能带我回忆一下那个过程？"
  → 追问："你当时在想什么？""那之后发生了什么？"

深度追问（5 min）— 挖掘未满足的需求
  "如果有一个魔法棒，你会改变这个过程里的什么？"
  
收尾（5 min）— 总结和感谢
  "我们聊了很多，我总结一下，你看对不对..."
```

### 好的问题 vs 坏的引导

| ❌ 引导性问题 | ✅ 开放式问题 |
|------------|------------|
| "你是不是觉得这个按钮太小了？" | "你上次找这个功能的时候，经历了什么？" |
| "你喜不喜欢我们的新设计？" | "和之前的版本相比，你觉得有什么不同？" |
| "你会用这个导出功能吗？" | "你上次需要导出数据的时候，是怎么做的？" |
| "为什么不买？是不是因为太贵？" | "你最后选择了什么替代方案？那个方案好在哪？" |

### 追问技巧

**五问法（Five Whys 适配版）：**
```
用户："我不喜欢这个功能。"
  → "能多说一点吗？具体是哪部分让你不喜欢？"
用户："太复杂了。"
  → "还记得上次你想用这个功能完成什么吗？"
用户："我想导出报表给老板看。"
  → "老板一般会关注报表里的哪些数据？"
用户："销量和利润率。"
  → "如果有一个方式能让老板自己看，你只需要分享一个链接，会更好吗？"
```

**沉默的力量：** 用户说完一句话后，默数 3 秒再开口。70% 的情况下用户会继续补充更深层的信息。

## 受访者招募

### 招募标准（Screener）

| 标准 | 目标 |
|------|------|
| 行为标准 | 过去 X 天使用过 Y 功能的用户 |
| 排除条件 | 行业内从业者、员工、亲友 |
| 多样性 | 性别、年龄、使用频率的均衡分布 |

### 样本量指南

| 研究类型 | 建议样本量 | 理由 |
|---------|----------|------|
| 探索性访谈 | 5-8 人 | 5 人后开始听到重复模式（Nielsen 定律） |
| 验证性访谈 | 8-12 人 | 需要覆盖更多细分用户 |
| 分群对比 | 每组 5-6 人 | 新用户 vs 老用户、付费 vs 免费 |

## 访谈后分析

### Affinity Mapping 方法

```
1. 转录（可选但推荐）→ 把录音转成文字
2. 提取关键引语 → 每句话一张便利贴（物理或 Miro/FigJam）
3. 分组 → 相似的观点放在一起
4. 命名主题 → 每组给一个总结性的名字
5. 投票 → 团队投票选出最重要的 3-5 个洞察
6. 写报告 → 不是所有观察都值得写，只写"如果不做就会出问题"的
```

## 质量原则

1. **不要问"你会不会用"** — 用户说的和做的是两回事。问他过去做了什么
2. **沉默是金** — 用户停顿时不要急着填话，沉默的 3 秒是信息的金矿
3. **每场 ≤ 1 小时** — 超过 45 分钟，受访者和主持人都开始疲惫
4. **记录原话不是记录总结** — "登录流程让我想砸电脑" ≠ "用户对登录体验不满"
5. **5 个人就够了** — 探索性访谈超过 8 人，边际收益急剧下降

## 使用示例

**用户输入：**
> 我们做了一个 B2B SaaS 的报表功能，但使用率很低。想找几个客户聊聊为什么不用。

**AI 执行过程：**
1. 明确研究目标：不是"为什么不报表"，是"客户现在怎么满足报表需求？报表功能在这个流程里卡在哪里？"
2. 设计漏斗式提纲：从日常工作 → 报表需求 → 我们的产品 → 替代方案
3. 准备追问链：每次用户说"不方便"时追问"上次不方便的时候你是怎么解决的？"
4. 输出完整讨论指南 + 受访者招募标准 + 分析框架
