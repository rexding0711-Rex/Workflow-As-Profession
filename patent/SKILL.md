# 专利分析（Patent Analysis）

## 核心能力

专利 = 技术护城河 + 竞争情报来源。
本 Skill 覆盖专利生命周期中的分析和策略环节，
不做法律意见书（最终应由专利律师确认），但能做 80% 的前期分析工作。

## 工作流

### Step 1：专利检索

**检索维度**：
- 技术关键词（中文 + 英文 + 同义词）
- IPC 分类号
- 申请人/专利权人
- 发明人
- 引用关系（被谁引用、引用了谁）

**检索策略**：
1. 从宽到窄：先用关键词泛搜，再按 IPC 缩小
2. 引文追踪：找到核心专利后，查其 forward/backward 引用
3. 同族专利：同一发明在不同国家的申请（优先看 PCT、US、CN、EP、JP）

### Step 2：专利地图

**技术景观图（Patent Landscape）**：
- X 轴：技术方向（如：材料/结构/工艺/应用）
- Y 轴：时间（申请年）
- 球大小：专利族数量
- 颜色：申请人

**空白点分析（White Space）**：
- 哪些技术方向专利密集（红海）？
- 哪些方向专利稀疏但市场前景好（蓝海）？
- 竞争对手的专利围墙在哪里？

### Step 3：FTO（Freedom to Operate）分析

**三步法**：
1. **识别风险专利**：在目标市场检索相关有效专利
2. **权利要求比对**：逐条比对已识别专利的权利要求 vs 我方产品/工艺
3. **规避设计**：如存在侵权风险，提出技术规避方案

**关键判断**：
- 专利是否在有效期内？（20 年保护期，从申请日起算）
- 专利是否在目标市场有效？（属地原则）
- 权利要求是否覆盖我方方案？（全要素规则——如果缺任一项权利要求的要素，不侵权）

### Step 4：专利策略

**布局策略**：
| 类型 | 目的 | 示例 |
|------|------|------|
| 基础专利 | 保护核心技术 | 新材料配方 |
| 外围专利 | 阻止竞争 | 改进工艺、应用场景 |
| 防御专利 | 防止被诉 | 竞争对手可能用到的技术 |
| 标准必要专利 | 收取许可费 | 纳入行业标准的必要技术 |

**申请决策矩阵**：
- 技术重要性 × 可检测性（能否从产品逆向？）
- 高重要性 + 高可检测 → 必须申请
- 高重要性 + 低可检测 → 考虑商业秘密保护
- 低重要性 + 高可检测 → 公开但不申请（阻止他人获得专利）
- 低重要性 + 低可检测 → 不申请

### Step 5：竞争情报

从专利看竞争对手：
- 研发方向（近年申请了什么？）
- 技术路线（选择了哪种技术方案？）
- 人才流动（发明人去了哪里？）
- 市场意图（在哪些国家布局？）
- 合作网络（共同申请、转让记录）

## 输出物清单
- 专利检索报告（关键词、检索式、结果统计）
- 专利地图（技术-时间-申请人三维分布）
- FTO 分析报告（风险专利列表、权利要求比对、规避建议）
- 专利布局策略（申请清单、优先级、预算估算）
- 竞争对手专利监控简报

## 注意事项
- 本 Skill 做分析辅助，不做法律意见书
- 关键专利决策必须由注册专利律师/代理人确认
- FTO 分析有地域性：专利只在注册国有效
- 临时申请（Provisional）不公开，检索不到是正常的
