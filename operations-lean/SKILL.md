# 精益运营 Skill

## 何时触发

用户请求：
- 优化业务流程或运营效率
- 降低生产成本或运营成本
- 提升交付速度或服务质量
- 诊断运营瓶颈
- "怎么把成本降30%" / "交付总是延期"

## 核心框架：精益五原则 + 瓶颈理论 + OEE

### 步骤1：价值流映射（Value Stream Mapping）

识别从原材料到客户手中的全部活动：

```
客户需求 → [活动1] → [活动2] → [活动3] → [活动4] → 交付客户
              ↑          ↑          ↑          ↑
           增值时间   等待时间   增值时间   等待时间
```

**分类**：
- **增值活动（VA）**：客户愿意付钱的活动（如加工、设计）
- **必要非增值（NNVA）**：不直接增值但必须做（如检验、审批）
- **浪费（NVA）**：完全不产生价值（如等待、返工、搬运）

**精益七浪费**：
1. 过量生产（生产早于需求）
2. 等待（人员/设备闲置）
3. 运输（不必要的搬运）
4. 过度加工（超出需求的精度）
5. 库存（原材料/在制品/成品积压）
6. 动作（不必要的人员动作）
7. 缺陷（返工/报废）

### 步骤2：瓶颈识别（TOC约束理论）

任何一个系统的产出，由瓶颈环节决定。

**识别方法**：
- 哪个环节的在制品堆积最多？
- 哪个环节的设备/人员利用率最高？
- 哪个环节的延期最严重？

**五步法**：
1. 识别瓶颈（找到约束）
2. 挖尽瓶颈（让瓶颈满负荷运转）
3. 迁就瓶颈（其他环节配合瓶颈节奏）
4. 提升瓶颈（增加瓶颈产能）
5. 如果瓶颈突破，回到步骤1（新的瓶颈会出现）

### 步骤3：流程效率评估（OEE）

设备综合效率 = 可用率 × 性能率 × 良品率

适用场景：制造业、数据中心、任何有固定产能的业务

- **可用率** = 实际运行时间 / 计划时间（故障、换型损失）
- **性能率** = 实际产出 / 理论产出（速度损失、小停机）
- **良品率** = 合格品 / 总产出（质量损失）

**世界级标准**：OEE > 85%（可用率90% × 性能率95% × 良品率99%）

### 步骤4：持续改善（Kaizen）

PDCA循环：

```
Plan（计划）→ Do（执行）→ Check（检查）→ Act（改进）
     ↑___________________________________________|
```

**Kaizen事件**：
- 聚焦一个具体问题
- 3-5天集中攻关
- 跨职能团队参与
- 目标：可量化的改善

### 步骤5：运营指标体系

分层指标：

| 层级 | 指标 | 频率 | 责任 |
|------|------|------|------|
| 战略层 | 收入、利润、市场份额 | 月/季 | 高管 |
| 战术层 | 订单周期、库存周转、客户满意度 | 周/月 | 中层 |
| 操作层 | 设备OEE、一次合格率、产出效率 | 日/班 | 一线 |

**关键原则**：
- 每个层级不超过5个核心指标
- 指标必须可实时获取
- 指标与激励机制挂钩

## 运营案例库

- **丰田生产方式**：JIT+看板+自働化（自动化+人性）
- **亚马逊履约中心**：算法调度+随机存储+按单拣选
- **富士康**：精益生产线+单元生产+快速换型
- **麦当劳**：标准化操作手册+秒表工程+特许经营
- **SHEIN**：小单快反+数字化供应链（TOC应用）

## 使用示例

**用户输入**：
> "我们工厂交付周期太长，客户总投诉，怎么优化？"

**AI执行**：
1. 先问关键数据：当前交付周期？瓶颈工序？在制品数量？
2. 绘制价值流图（用户描述版）
3. 识别瓶颈和浪费
4. 给出TOC五步法建议
5. 设计改善路线图
