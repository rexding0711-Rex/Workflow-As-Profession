# 工作坊引导（Workshop Facilitation）

## 核心能力

工作坊是知识工作中最高频的协作方式。本 Skill 覆盖从设计到交付的完整流程，
不做"会议记录"，而是做"结构化引导"。

## 工作流

### Step 1：需求诊断
- 明确工作坊目的（对齐？脑暴？决策？培训？变革？）
- 识别参与方（谁需要在场？各自的立场和诉求？）
- 确定产出物（决策清单？方案框架？行动计划？）

### Step 2：议程设计
按目的选择结构化框架：

**战略对齐型**（SOS — Start/Obstacles/Solutions）：
1. 现状共识（30min）→ 2. 障碍识别（45min）→ 3. 方案生成（60min）→ 4. 优先级投票（30min）

**创意脑暴型**（Crazy 8s + Dot Voting）：
1. 问题定义（15min）→ 2. 发散草图（8分钟8个想法）→ 3. 方案收敛（点投）→ 4. 概念深化（45min）

**变革沟通型**（ADKAR Workshop）：
1. Awareness 建立紧迫感 → 2. Desire 激发意愿 → 3. Knowledge 知识传递 → 4. Ability 能力演练 → 5. Reinforcement 巩固机制

**培训赋能型**（Tell-Show-Do-Review）：
1. 概念讲解（Tell, 20%）→ 2. 示范演示（Show, 20%）→ 3. 实操演练（Do, 50%）→ 4. 复盘反馈（Review, 10%）

### Step 3：引导执行
- 开场：明确目标、议程、规则（停车场规则：无关话题先记下）
- 过程：时间盒管理、确保每位参与、记录关键分歧
- 收尾：决策清单、行动项（谁/做什么/截止日期）、下次跟进节点

### Step 4：产出沉淀
- 工作坊记录（关键讨论、分歧点、决策依据）
- 行动项清单（Owner + Deadline）
- 后续跟进计划

## 常见场景速查

| 场景 | 推荐框架 | 时长 | 典型产出 |
|------|---------|------|---------|
| 高管战略对齐 | SOS | 3h | 战略方向共识、优先级清单 |
| 产品脑暴 | Crazy 8s | 2h | 概念草图 x N、优先级矩阵 |
| 变革动员 | ADKAR | 3h | 变革路线图、沟通计划 |
| 团队培训 | Tell-Show-Do-Review | 2-4h | 培训记录、能力评估 |
| 跨部门协作 | World Café | 3h | 共识声明、协作协议 |
| 问题根因分析 | Fishbone + 5 Whys | 2h | 根因分析图、改进措施 |

## 注意事项
- 工作坊 ≠ 会议，必须有明确产出物
- 引导者不参与内容讨论，只管理流程
- 停车场规则很重要：跑题的话题先记下，会后处理
- 超过 3 小时的工作坊必须安排中间休息
- 远程工作坊需要提前测试工具（白板、投票、分组讨论）
