# Summary Writing Guide

> For sub-agents: When there are clear content sources (prerequisite documents, or complete information passed by the main agent in the task description), you need to write presentation pages based on these contents.

## Applicable Scenarios
- Prior workflows have produced documentation (papers, reports, prior workflow artifacts, etc.), and the main agent assigns you to read these documents
- The main agent has passed complete content such as search results or research information directly into your task description

## Workflow

### 1. Understand Page Direction
- Read the `outline.md` produced by the main agent to understand the title, content direction, and page type for each page you are responsible for
- Clarify the core information point each page should convey

### 2. Read Content Sources
- If the main agent specifies prerequisite documents: Read these documents completely and extract content corresponding to each page
- If the main agent passes complete information in the task description: Carefully read the provided content and map it to the page directions in outline.md
- Mark key data and core arguments to ensure accurate citation during subsequent writing

### 3. Content Writing Principles
1. **Faithful to sources**: Distill rather than rewrite; do not add viewpoints or data that do not exist in the sources
2. **Per-page focus**: Each page should focus on one clear information point; avoid content piling
3. **Data precision**: Preserve specific numbers, percentages, years, and other data precision from the sources; do not obscure them
4. **Key support**: Use key data and cases from the sources to support page viewpoints
5. **Moderate condensation**: Condense lengthy discussions into concise expressions suitable for presentations, without losing core information
